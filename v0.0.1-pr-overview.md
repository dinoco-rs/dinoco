# v0.0.1 Overview

## Summary

The `v0.0.1` release marks the first public delivery of the Dinoco ecosystem, establishing the foundation for schema-driven database development with a Rust-first workflow. This version brings together the ORM, engine, CLI, code generation, and documentation into a cohesive flow that already supports local project bootstrap, schema modeling, migration generation and execution, and Rust model generation.

## Highlights

- First public release of the Dinoco stack, including `dinoco`, `dinoco_engine`, `dinoco_cli`, `dinoco_codegen`, `dinoco_compiler`, `dinoco_derives`, and `dinoco_formatter`.
- End-to-end schema workflow with local initialization, schema compilation, migration generation, migration execution, schema restoration, and generated Rust models.
- Multi-database adapter support for `SQLite`, `PostgreSQL`, and `MySQL`.
- Type-safe query and persistence foundations covering CRUD operations, filters, ordering, pagination, relation loading, relation mutations, and relation counting.
- Migration infrastructure with history tracking, local migration artifacts, and database synchronization flows through the CLI.
- Initial public documentation for `v0.0.1`, covering ORM concepts, schema modeling, client usage, and CLI commands.

## CLI Scope in v0.0.1

The CLI now supports the core daily workflow for Dinoco projects:

- `dinoco init` to bootstrap a project with the recommended structure and database configuration
- `dinoco migrate generate` to create migration artifacts from schema changes
- `dinoco migrate generate --apply` to generate and apply changes immediately
- `dinoco migrate run` to execute pending local migrations
- `dinoco models generate` to regenerate Rust models from the latest persisted migration state
- `dinoco schema restore` to rebuild `schema.dinoco` from migration metadata
- `dinoco database reset` to clear the configured database in development flows

## Engine and ORM Scope in v0.0.1

This release establishes the runtime and query foundations of Dinoco:

- Query builder coverage for filtering, ordering, pagination, batched updates, deletes, counts, and window-function-related SQL generation
- Cross-adapter support for relation workflows, including one-to-many and many-to-many mutation flows
- Constraint handling and adapter-specific behavior across supported databases
- Migration planning and SQL generation for schema evolution, including SQLite-specific rebuild paths where needed

## Reliability Notes

This version also includes hardening in migration behavior for `sqlite` flows. When a generated migration fails during apply/run, the CLI should not leave the new migration persisted as if it were successful, and it should not mark that migration as applied in history. This keeps local artifacts and migration state aligned with the actual database result.

## Documentation

The `v0.0.1` documentation is the first public documentation set for Dinoco and introduces:

- Dinoco ORM overview
- Client installation and querying basics
- Schema modeling and generators
- CLI installation and command references
- Migration and Studio entry points

## Notes

- `v0.0.1` is the baseline public release and focuses on delivering the core platform workflow with test coverage across the main adapters and CLI flows.
- Some product areas, such as Dinoco Supersonic and DinocoDB, are introduced in the version structure but are still marked as under construction.
