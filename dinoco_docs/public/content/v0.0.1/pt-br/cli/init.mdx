# Como usar?

O comando `dinoco init` inicializa o ambiente Dinoco no projeto atual.

Ele prepara a base necessária para começar a trabalhar com schema, migrações e geração de modelos.

---

## O que o comando faz

Ao executar `dinoco init`, a CLI guia a configuração inicial do ambiente Dinoco.

Em geral, este fluxo prepara:

- Estrutura base do diretório `dinoco`
- Arquivos de schema iniciais
- Configuração do banco de dados
- Base para futuras migrações

## Quando usar

Use este comando quando:

- Você estiver configurando o Dinoco pela primeira vez no projeto
- Você quiser criar a estrutura inicial do ambiente local
- Você precisar preparar o projeto para gerar migrações e modelos

## Próximos passos

Após `init`, o fluxo mais comum é:

```bash
dinoco migrate generate
```

ou, se você quiser gerar e aplicar imediatamente:

```bash
dinoco migrate generate --apply
```
