# Перечисления

Перечисления позволяют ограничить поле фиксированным набором известных значений в схеме Dinoco.

Они полезны, когда значение должно быть предсказуемым, проверенным и повторно используемым между моделями.

---

## Что такое перечисление

`enum` определяет закрытый список возможных значений.

```dinoco
enum Role {
	USER
	ADMIN
}
```

В этом случае `Role` может принимать только `USER` или `ADMIN`.

## Использование в моделях

После определения перечисление может быть использовано как тип поля в любой модели.

```dinoco
enum Role {
	USER
	ADMIN
}

model User {
	id   Integer @id @default(autoincrement())
	role Role    @default(USER)
}
```

Здесь:

- `role` использует перечисление `Role`.
- `@default(USER)` определяет значение поля по умолчанию.

## Когда использовать перечисления

Перечисления полезны для представления таких значений, как:

- Роли пользователя
- Статусы публикации
- Этапы рабочего процесса
- Ситуации оплаты

Пример:

```dinoco
enum PostStatus {
	DRAFT
	REVIEW
	PUBLISHED
	ARCHIVED
}

model Post {
	id     Integer    @id @default(autoincrement())
	title  String
	status PostStatus @default(DRAFT)
}
```

## Рекомендации

- Используйте перечисления, когда возможные значения известны и конечны.
- Предпочитайте имена в PascalCase для перечислений и значения в UPPER_CASE.
- Используйте `@default(...)`, когда есть естественное начальное состояние.
