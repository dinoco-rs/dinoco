export const title = "insert_into";

# insert_into

Usato per inserire un record.

---

## Cosa puoi fare

- Passare valori con `.values(item)`
- Inserire con relazione tramite `.with_relation(related)`
- Connettere una relazione esistente tramite `.with_connection(connected)`
- Eseguire con `.execute(&client)`

## Descrizione dei metodi

- `.values(item)`: definisce il record che verrà inserito.
- `.with_relation(related)`: inserisce un nuovo record correlato insieme al padre.
- `.with_connection(connected)`: connette il record inserito a un elemento già esistente, normalmente in flussi di relazione supportati per la connessione.
- `.returning::&lt;T&gt;()`: cambia il ritorno a una proiezione tipizzata dell'elemento inserito.
- `.execute(&client)`: esegue la scrittura nel database.

## Ritorno

Senza `.returning::&lt;T&gt;()`, il ritorno è:

```rust
DinocoResult<()>
```

Con `.returning::&lt;T&gt;()`, il ritorno diventa:

```rust
DinocoResult<T>
```

## Esempio semplice

```rust
dinoco::insert_into::<User>()
    .values(User {
        id: "usr_1".to_string(),
        email: "ana@acme.com".to_string(),
        name: "Ana".to_string(),
    })
    .execute(&client)
    .await?;
```

## Esempio con relazione

Usa `.with_relation(...)` quando il modello generato supporta l'inserimento del padre e del correlato insieme.

```rust
dinoco::insert_into::<User>()
    .values(user)
    .with_relation(profile)
    .execute(&client)
    .await?;
```

## Esempio con connessione

Usa `.with_connection(...)` quando vuoi inserire un elemento e connettere una relazione già esistente.

```rust
dinoco::insert_into::<User>()
    .values(new_user)
    .with_connection(existing_team)
    .execute(&client)
    .await?;
```

## Esempio con ritorno tipizzato

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(User)]
struct UserSummary {
    id: i64,
    name: String,
}

let created = dinoco::insert_into::<User>()
    .values(User { id: 1, name: "Matheus".to_string() })
    .returning::<UserSummary>()
    .execute(&client)
    .await?;
```
