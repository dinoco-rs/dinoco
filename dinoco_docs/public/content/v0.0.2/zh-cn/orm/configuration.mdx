# 配置

`config {}` 块定义了 Dinoco 如何连接到数据库以及项目中可用的额外功能。

## `config {}` 中包含什么

- `database`：定义主适配器，例如 `sqlite`、`postgresql` 或 `mysql`。
- `database_url`：提供主连接 URL。
- `read_replicas`：可选，添加读取副本。
- `redis`：可选，启用基于 Redis 的缓存和队列。

## 基本示例

```dinoco
config {
    database = "sqlite"
    database_url = env("DATABASE_URL")
}
```

## Redis 示例

```dinoco
config {
    database = "postgresql"
    database_url = env("DATABASE_URL")

    redis = {
        url = env("REDIS_URL")
    }
}
```

## 副本示例

```dinoco
config {
    database = "postgresql"
    database_url = env("DATABASE_URL")

    read_replicas = [
        env("DATABASE_READ_URL_1"),
        env("DATABASE_READ_URL_2")
    ]
}
```

## 生成的 API 有何变化

- 启用 `redis` 后，项目会暴露辅助函数，例如 `client.cache()`、`.cache(...)` 以及带有 `workers()` 的队列。
- 启用 `read_replicas` 后，读取操作可以使用副本，而写入操作仍继续在主数据库上进行。
- 如果没有 `redis`，缓存和队列功能将无法完全使用。

## 何时编辑此块

- 更改项目的主数据库时。
- 添加 Redis 用于缓存或队列时。
- 配置读取副本时。
- 调整连接环境变量时。
