# Comment utiliser ?

La commande `dinoco migrate run` exécute toutes les migrations en attente.

Elle est utilisée pour aligner la base de données configurée avec l'historique des migrations déjà générées.

---

## Ce que fait la commande

Lors de l'exécution de cette commande, le CLI :

- Lit l'historique des migrations
- Vérifie celles qui n'ont pas encore été appliquées
- Exécute les migrations en attente dans l'ordre

## Quand l'utiliser

Utilisez cette commande lorsque :

- Vous avez déjà des migrations générées
- Vous voulez appliquer les changements en attente dans la base de données
- Vous préparez un environnement local ou de déploiement

## Prochaines étapes

Si nécessaire, générez les modèles avec :

```bash
dinoco models generate
```
