# 模型

`模型`定义了您的应用程序在 Dinoco 模式中的核心实体。每个 `模型` 通常代表数据库中的一个表，并作为代码生成、类型化查询以及与 Dinoco API 操作的基础。

---

## 模型代表什么

一个 `模型` 描述：

- 实体的名称。
- 存储在数据库中的字段。
- 哪些字段是必需的或可选的。
- 哪些字段是唯一的或标识符。
- 这些数据将如何被代码生成器和 API 使用。

示例：

```dinoco
model User {
	id    Integer @id @default(autoincrement())
	email String  @unique
	name  String?
}
```

在这个例子中：

- `User` 是模型。
- `id`、`email` 和 `name` 是标量字段。
- `id` 是主标识符。
- `email` 具有唯一性约束。

## 完整示例

一个带有模型的简单模式通常看起来像这样：

```dinoco
config {
	database = "postgresql"
	database_url = env("DATABASE_URL")
}

model User {
	id        Integer  @id @default(autoincrement())
	email     String   @unique
	name      String?
	active    Boolean  @default(true)
	createdAt DateTime @default(now())
}

model Post {
	id        Integer  @id @default(autoincrement())
	title     String
	content   String?
	published Boolean  @default(false)
	createdAt DateTime @default(now())
}
```

## 字段结构

模型的每个字段都由以下部分组成：

- 名称
- 类型
- 可选修饰符
- 可选属性

示例：

```dinoco
email String @unique
```

在这一行中：

- `email` 是字段名称。
- `String` 是类型。
- `@unique` 是一个属性。

## 字段类型

字段可以表示模式中的基本值，如文本、数字、布尔值和日期。

### 标量字段

这些字段存储直接值，如文本、数字、布尔值和日期。

```dinoco
model Product {
	id          Integer  @id @default(autoincrement())
	name        String
	description String?
	price       Float
	active      Boolean  @default(true)
	createdAt   DateTime @default(now())
}
```

## 类型修饰符

Dinoco 支持两个主要修饰符：

| 修饰符 | 含义       | 示例          |
| :----- | :--------- | :------------ |
| `?`    | 可选字段   | `name String?`  |
| `[]`   | 列表       | `tags String[]` |

### 可选字段

```dinoco
model User {
	id   Integer @id @default(autoincrement())
	name String?
}
```

`name` 可以为 null 或缺失，具体取决于数据库和生成的层。

### 列表字段

```dinoco
model Article {
	id   Integer  @id @default(autoincrement())
	tags String[]
}
```

当数据库和流程支持这种结构类型时，此格式表示一个值列表。

## 最常见的属性

属性会改变字段和模型的行为。

| 属性            | 用途                         |
| :-------------- | :--------------------------- |
| `@id`           | 定义主标识符                 |
| `@default(...)` | 定义默认值                   |
| `@unique`       | 保证唯一性                   |

### `@id`

定义唯一标识记录的字段。

```dinoco
id Integer @id @default(autoincrement())
```

每个模型都必须有一个明确的标识符，以便生成的 API 能够安全地操作。

### `@default(...)`

为字段定义一个默认值。

```dinoco
active    Boolean  @default(true)
createdAt DateTime @default(now())
id        Integer  @default(autoincrement())
```

常用函数和值：

| 示例                        | 用途            |
| :-------------------------- | :-------------- |
| `@default(false)`           | 默认布尔值      |
| `@default(now())`           | 当前日期        |
| `@default(autoincrement())` | 增量整数        |
| `@default(uuid())`          | UUID 标识符     |

### `@unique`

确保字段值不重复。

```dinoco
model User {
	id    Integer @id @default(autoincrement())
	email String  @unique
}
```

此属性非常适合电子邮件、用户名和外部代码等字段。

## 模型装饰器

除了单个字段的属性外，Dinoco 还支持应用于整个模型块的装饰器。

| 装饰器                | 用途                                |
| :-------------------- | :---------------------------------- |
| `@@ids([...])`        | 定义复合主键                        |
| `@@table_name("...")` | 映射数据库中表的实际名称            |

### `@@ids([...])`

当记录的身份依赖于多个字段时，使用 `@@ids`。

```dinoco
model Membership {
	userId Integer
	teamId Integer
	role   String

	@@ids([userId, teamId])
}
```

此格式在关联表和自然唯一性已是复合的场景中非常有用。

### `@@table_name("...")`

当您希望在模式中保持一个更友好的模型名称，但映射到数据库中的另一个物理名称时，使用 `@@table_name()`。

```dinoco
model User {
	id    Integer @id @default(autoincrement())
	email String  @unique

	@@table_name("users")
}
```

在这种情况下：

- 模型在模式和生成的 API 中仍然称为 `User`。
- 数据库中的物理表变为 `users`。

## 用户模型示例

```dinoco
model User {
	id        Integer  @id @default(autoincrement())
	email     String   @unique
	name      String?
	active    Boolean  @default(true)
	createdAt DateTime @default(now())
}
```

代码生成后，此模型可以直接与 Dinoco API 一起使用。

## 使用 Dinoco API 搜索用户的示例

### 搜索单个记录

```rust
let user = dinoco::find_first::<User>()
    .cond(|x| x.id.eq(1_i64))
    .execute(&client)
    .await?;
```

### 搜索多个记录

```rust
let users = dinoco::find_many::<User>()
    .cond(|x| x.name.includes("Ana"))
    .order_by(|x| x.id.asc())
    .take(10)
    .execute(&client)
    .await?;
```

## 使用 Dinoco API 创建用户的示例

```rust
dinoco::insert_into::<User>()
    .values(User {
        id: 0,
        email: "bia@dinoco.rs".to_string(),
        name: Some("Bia".to_string()),
        active: true,
        createdAt: dinoco::Utc::now(),
    })
    .execute(&client)
    .await?;
```

## 使用 Dinoco API 更新用户的示例

```rust
dinoco::update::<User>()
    .cond(|x| x.id.eq(1_i64))
    .values(User {
        id: 1,
        email: "bia@dinoco.rs".to_string(),
        name: Some("Beatriz".to_string()),
        active: true,
        createdAt: dinoco::Utc::now(),
    })
    .execute(&client)
    .await?;
```

如果您想对单个字段进行原子更新，`find_and_update` 流程通常更直接：

```rust
let user = dinoco::find_and_update::<User>()
    .cond(|x| x.id.eq(1_i64))
    .update(|x| x.name.set("Beatriz"))
    .execute(&client)
    .await?;
```

## 使用 Dinoco API 删除用户的示例

```rust
dinoco::delete::<User>()
    .cond(|x| x.id.eq(1_i64))
    .execute(&client)
    .await?;
```

批量删除：

```rust
dinoco::delete_many::<User>()
    .cond(|x| x.active.eq(false))
    .execute(&client)
    .await?;
```

## 快速摘要

| 概念       | 示例                   | 目标                      |
| :--------- | :--------------------- | :------------------------ |
| 模型       | `model User { ... }`   | 表示一个实体              |
| 标量字段   | `email String`         | 存储简单值                |
| 可选字段   | `name String?`         | 允许值缺失                |
| 列表字段   | `tags String[]`        | 存储多个值                |
| ID         | `id Integer @id`       | 唯一标识                  |
| 默认值     | `@default(now())`      | 自动填充                  |
| 唯一性     | `email String @unique` | 避免重复                  |

## 何时创建新模型

当您的应用程序中的一个实体需要满足以下条件时，您通常会创建一个新的 `模型`：

- 需要持久化到数据库中。
- 具有自己的身份。
- 可以独立查询。
- 具有自己的读写规则。

常见示例：

- `User`
- `Post`
- `Comment`
- `Category`
- `Order`
- `Invoice`
