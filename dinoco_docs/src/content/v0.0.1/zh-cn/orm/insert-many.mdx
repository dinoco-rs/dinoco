export const title = "insert_many";

# insert_many

用于批量插入。

---

## 你可以做什么

- `.values(Vec<M>)`: 定义要批量插入的记录。
- `.with_relation(Vec<R>)`: 为每个父元素插入一个新的关联实体。
- `.with_relations(Vec<Vec<R>>)` : 为每个父元素插入多个新的关联实体。
- `.with_connection(Vec<R>)`: 为每个插入的父元素连接一个已存在的关联。
- `.with_connections(Vec<Vec<R>>)` : 为每个插入的父元素连接多个已存在的关联，通常用于多对多场景。
- `.returning::<T>()`: 将返回值更改为插入元素的类型化列表。
- `.execute(&client)`: 执行批量操作。

## 返回值

如果不使用 `.returning::<T>()`，返回值是：

```rust
DinocoResult<()>
```

如果使用 `.returning::<T>()`，返回值是：

```rust
DinocoResult<Vec<T>>
```

## 简单示例

```rust
dinoco::insert_many::<User>()
    .values(vec![
        User { id: "u1".into(), email: "a@acme.com".into(), name: "A".into() },
        User { id: "u2".into(), email: "b@acme.com".into(), name: "B".into() },
    ])
    .execute(&client)
    .await?;
```

## 带返回值的示例

```rust
let created = dinoco::insert_many::<User>()
    .values(vec![
        User { id: 2, name: "Ana".to_string() },
        User { id: 3, name: "Caio".to_string() },
    ])
    .returning::<User>()
    .execute(&client)
    .await?;
```

## `with_relation(...)` 示例

当每个父元素都需要创建并关联时，请使用 `with_relation(...)`。

```rust
dinoco::insert_many::<Post>()
    .values(vec![
        Team { id: "team-11".into(), name: "Data".into() },
        Team { id: "team-12".into(), name: "DX".into() },
    ])
    .with_relation(vec![
        Member { id: "member-11".into(), name: "Rafa".into(), teamId: "legacy".into() },
        Member { id: "member-12".into(), name: "Bia".into(), teamId: "legacy".into() },
    ])
    .execute(&client)
    .await?;
```

## `with_relations(...)` 示例

当每个父记录有多个子记录时，请使用 `with_relations(...)`；这些记录将自动创建并关联。

```rust
dinoco::insert_many::<Post>()
    .values(vec![
        Team { id: "team-11".into(), name: "Data".into() },
        Team { id: "team-12".into(), name: "DX".into() },
    ])
    .with_relation(vec![
        vec![
			Member { id: "member-11".into(), name: "Rafa".into(), teamId: "legacy".into() },
        	Member { id: "member-12".into(), name: "Bia".into(), teamId: "legacy".into() },
		],

		vec![Member { id: "member-11".into(), name: "Rafa".into(), teamId: "legacy".into() }]
    ])
    .execute(&client)
    .await?;
```

## `with_connection(...)` 示例

当每个父元素需要连接到一个已存在的记录时，请使用 `with_connection(...)`。

```rust
dinoco::insert_many::<Team>()
    .values(vec![
        Team { id: "team-11".into(), name: "Data".into() },
        Team { id: "team-12".into(), name: "DX".into() },
    ])
    .with_connection(vec![
        Member { id: "member-11".into(), name: "Rafa".into(), teamId: "legacy".into() },
        Member { id: "member-12".into(), name: "Bia".into(), teamId: "legacy".into() },
    ])
    .execute(&client)
    .await?;
```

## `with_connections(...)` 示例

当每个父元素需要连接到多个已存在的记录时，请使用 `with_connections(...)`。
这在多对多关系中很常见。

```rust
dinoco::insert_many::<Article>()
    .values(vec![
        Article { id: "article-11".into(), title: "Connect Multiple".into() },
        Article { id: "article-12".into(), title: "Connect Batch".into() },
    ])
    .with_connections(vec![
        vec![
            Label { id: "label-11".into(), name: "orm".into() },
            Label { id: "label-12".into(), name: "rust".into() },
        ],
        vec![
            Label { id: "label-10".into(), name: "backend".into() },
        ],
    ])
    .execute(&client)
    .await?;
```

## 说明

- 当父元素在内存中已有已知 ID 时，`with_relation(...)` 和 `with_relations(...)` 效果最佳。
- `with_connection(...)` 和 `with_connections(...)` 用于连接已存在的关联。
- `with_connection(...)` 需要与 `values(...)` 相同数量的元素。
- `with_connections(...)` 需要与主向量相同数量的组。
