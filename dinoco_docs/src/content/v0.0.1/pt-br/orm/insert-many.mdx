export const title = "insert_many";

# insert_many

Usado para inserção em lote.

---

## O que você pode fazer

- `.values(Vec<M>)`: define os registros que serão inseridos em lote.
- `.with_relation(Vec<R>)`: insere exatamente um relacionado novo para cada pai.
- `.with_relations(Vec<Vec<R>>)` : insere vários relacionados novos para cada pai.
- `.with_connection(Vec<R>)`: conecta exatamente uma relação já existente para cada pai inserido.
- `.with_connections(Vec<Vec<R>>)` : conecta várias relações já existentes para cada pai inserido, normalmente em cenários many-to-many.
- `.returning::<T>()`: muda o retorno para uma lista tipada dos itens inseridos.
- `.execute(&client)`: executa a operação em lote.

## Retorno

Sem `.returning::<T>()`, o retorno é:

```rust
DinocoResult<()>
```

Com `.returning::<T>()`, o retorno passa a ser:

```rust
DinocoResult<Vec<T>>
```

## Exemplo simples

```rust
dinoco::insert_many::<User>()
    .values(vec![
        User { id: "u1".into(), email: "a@acme.com".into(), name: "A".into() },
        User { id: "u2".into(), email: "b@acme.com".into(), name: "B".into() },
    ])
    .execute(&client)
    .await?;
```

## Exemplo com retorno

```rust
let created = dinoco::insert_many::<User>()
    .values(vec![
        User { id: 2, name: "Ana".to_string() },
        User { id: 3, name: "Caio".to_string() },
    ])
    .returning::<User>()
    .execute(&client)
    .await?;
```

## Exemplo com with_relation(...)

Use `with_relation(...)` quando cada pai precisa ser criado e conectado.

```rust
dinoco::insert_many::<Post>()
    .values(vec![
        Team { id: "team-11".into(), name: "Data".into() },
        Team { id: "team-12".into(), name: "DX".into() },
    ])
    .with_relation(vec![
        Member { id: "member-11".into(), name: "Rafa".into(), teamId: "legacy".into() },
        Member { id: "member-12".into(), name: "Bia".into(), teamId: "legacy".into() },
    ])
    .execute(&client)
    .await?;
```

## Exemplo com with_relations(...)

Use with_relations(...) quando cada registro pai tiver mais de um filho; os registros serão criados e vinculados automaticamente.

```rust
dinoco::insert_many::<Post>()
    .values(vec![
        Team { id: "team-11".into(), name: "Data".into() },
        Team { id: "team-12".into(), name: "DX".into() },
    ])
    .with_relation(vec![
        vec![
			Member { id: "member-11".into(), name: "Rafa".into(), teamId: "legacy".into() },
        	Member { id: "member-12".into(), name: "Bia".into(), teamId: "legacy".into() },
		],

		vec![Member { id: "member-11".into(), name: "Rafa".into(), teamId: "legacy".into() }]
    ])
    .execute(&client)
    .await?;
```

## Exemplo com with_connection(...)

Use `with_connection(...)` quando cada pai precisa ser conectado a exatamente um registro já existente.

```rust
dinoco::insert_many::<Team>()
    .values(vec![
        Team { id: "team-11".into(), name: "Data".into() },
        Team { id: "team-12".into(), name: "DX".into() },
    ])
    .with_connection(vec![
        Member { id: "member-11".into(), name: "Rafa".into(), teamId: "legacy".into() },
        Member { id: "member-12".into(), name: "Bia".into(), teamId: "legacy".into() },
    ])
    .execute(&client)
    .await?;
```

## Exemplo com with_connections(...)

Use `with_connections(...)` quando cada pai precisa ser conectado a vários registros já existentes.
Esse é um uso comum em relações many-to-many.

```rust
dinoco::insert_many::<Article>()
    .values(vec![
        Article { id: "article-11".into(), title: "Connect Multiple".into() },
        Article { id: "article-12".into(), title: "Connect Batch".into() },
    ])
    .with_connections(vec![
        vec![
            Label { id: "label-11".into(), name: "orm".into() },
            Label { id: "label-12".into(), name: "rust".into() },
        ],
        vec![
            Label { id: "label-10".into(), name: "backend".into() },
        ],
    ])
    .execute(&client)
    .await?;
```

## Observações

- `with_relation(...)` e `with_relations(...)` funcionam melhor quando o pai já possui um id conhecido em memória.
- `with_connection(...)` e `with_connections(...)` são voltados para conexão de relações já existentes.
- `with_connection(...)` exige o mesmo número de itens de `values(...)`.
- `with_connections(...)` exige o mesmo número de grupos em relação ao vetor principal.
