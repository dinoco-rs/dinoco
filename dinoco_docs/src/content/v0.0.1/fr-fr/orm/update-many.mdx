export const title = "update_many";

# update_many

Utilisé pour mettre à jour plusieurs enregistrements simultanément.

---

## Ce que vous pouvez faire

- `.cond(...)`: restreint les enregistrements qui peuvent être mis à jour.
- `.values(Vec<M>)`: définit les éléments utilisés dans la mise à jour par lot.
- `.returning::<T>()`: renvoie les enregistrements mis à jour sous forme de liste typée.
- `.execute(&client)`: exécute la mise à jour par lot.

## Retour

Sans `.returning::<T>()`, le retour est :

```rust
DinocoResult<()>
```

Avec `.returning::<T>()`, le retour est :

```rust
DinocoResult<Vec<T>>
```

## Exemple de base

```rust
dinoco::update_many::<User>()
    .values(vec![
        User { id: 1, email: "a@acme.com".into(), name: "Ana".into() },
        User { id: 2, email: "b@acme.com".into(), name: "Bia".into() },
    ])
    .execute(&client)
    .await?;
```

<h2>Exemple avec retour</h2>

```rust
let updated = dinoco::update_many::<User>()
    .values(vec![
        User { id: 2, name: "Ana Batch".to_string() },
        User { id: 3, name: "Caio Batch".to_string() },
    ])
    .returning::<User>()
    .execute(&client)
    .await?;
```

<h2>Exemple avec filtre</h2>

```rust
dinoco::update_many::<User>()
    .cond(|x| x.active.eq(true))
    .values(vec![
        User { id: 10, email: "a@acme.com".into(), name: "Ana".into() },
        User { id: 11, email: "b@acme.com".into(), name: "Bia".into() },
    ])
    .execute(&client)
    .await?;
```
