# insert_many

Utilisé pour l'insertion par lots.

---

## Ce que vous pouvez faire

- `.values(Vec&lt;M&gt;)`: définit les enregistrements qui seront insérés par lots.
- `.with_relation(Vec&lt;R&gt;)`: insère exactement un nouvel élément lié pour chaque parent.
- `.with_relations(Vec&lt;Vec\<R&gt;\>)` : insère plusieurs nouveaux éléments liés pour chaque parent.
- `.with_connection(Vec&lt;R&gt;)`: connecte exactement une relation existante pour chaque parent inséré.
- `.with_connections(Vec&lt;Vec\<R&gt;\>)` : connecte plusieurs relations existantes pour chaque parent inséré, généralement dans des scénarios many-to-many.
- `.returning::&lt;T&gt;()`: modifie le retour en une liste typée des éléments insérés.
- `.execute(&client)`: exécute l'opération par lots.

## Retour

Sans `.returning::&lt;T&gt;()`, le retour est :

```rust
DinocoResult<()>
```

Avec `.returning::&lt;T&gt;()`, le retour devient :

```rust
DinocoResult<Vec<T>>
```

## Exemple simple

```rust
dinoco::insert_many::<User>()
    .values(vec![
        User { id: "u1".into(), email: "a@acme.com".into(), name: "A".into() },
        User { id: "u2".into(), email: "b@acme.com".into(), name: "B".into() },
    ])
    .execute(&client)
    .await?;
```

## Exemple avec retour

```rust
let created = dinoco::insert_many::<User>()
    .values(vec![
        User { id: 2, name: "Ana".to_string() },
        User { id: 3, name: "Caio".to_string() },
    ])
    .returning::<User>()
    .execute(&client)
    .await?;
```

## Exemple avec with_relation(...)

Utilisez `with_relation(...)` lorsque chaque parent doit être créé et connecté.

```rust
dinoco::insert_many::<Post>()
    .values(vec![
        Team { id: "team-11".into(), name: "Données".into() },
        Team { id: "team-12".into(), name: "DX".into() },
    ])
    .with_relation(vec![
        Member { id: "member-11".into(), name: "Rafa".into(), teamId: "legacy".into() },
        Member { id: "member-12".into(), name: "Bia".into(), teamId: "legacy".into() },
    ])
    .execute(&client)
    .await?;
```

## Exemple avec with_relations(...)

Utilisez with_relations(...) lorsque chaque enregistrement parent a plus d'un enfant ; les enregistrements seront créés et liés automatiquement.

```rust
dinoco::insert_many::<Post>()
    .values(vec![
        Team { id: "team-11".into(), name: "Données".into() },
        Team { id: "team-12".into(), name: "DX".into() },
    ])
    .with_relation(vec![
        vec![
			Member { id: "member-11".into(), name: "Rafa".into(), teamId: "legacy".into() },
        	Member { id: "member-12".into(), name: "Bia".into(), teamId: "legacy".into() },
		],

		vec![Member { id: "member-11".into(), name: "Rafa".into(), teamId: "legacy".into() }]
    ])
    .execute(&client)
    .await?;
```

## Exemple avec with_connection(...)

Utilisez `with_connection(...)` lorsque chaque parent doit être connecté à exactement un enregistrement existant.

```rust
dinoco::insert_many::<Team>()
    .values(vec![
        Team { id: "team-11".into(), name: "Données".into() },
        Team { id: "team-12".into(), name: "DX".into() },
    ])
    .with_connection(vec![
        Member { id: "member-11".into(), name: "Rafa".into(), teamId: "legacy".into() },
        Member { id: "member-12".into(), name: "Bia".into(), teamId: "legacy".into() },
    ])
    .execute(&client)
    .await?;
```

## Exemple avec with_connections(...)

Utilisez `with_connections(...)` lorsque chaque parent doit être connecté à plusieurs enregistrements existants.
C'est une utilisation courante dans les relations many-to-many.

```rust
dinoco::insert_many::<Article>()
    .values(vec![
        Article { id: "article-11".into(), title: "Connecter Plusieurs".into() },
        Article { id: "article-12".into(), title: "Connecter par Lots".into() },
    ])
    .with_connections(vec![
        vec![
            Label { id: "label-11".into(), name: "orm".into() },
            Label { id: "label-12".into(), name: "rust".into() },
        ],
        vec![
            Label { id: "label-10".into(), name: "backend".into() },
        ],
    ])
    .execute(&client)
    .await?;
```

## Remarques

- `with_relation(...)` et `with_relations(...)` fonctionnent mieux lorsque le parent possède déjà un ID connu en mémoire.
- `with_connection(...)` et `with_connections(...)` sont destinés à la connexion de relations existantes.
- `with_connection(...)` exige le même nombre d'éléments que `values(...)`.
- `with_connections(...)` exige le même nombre de groupes par rapport au vecteur principal.
