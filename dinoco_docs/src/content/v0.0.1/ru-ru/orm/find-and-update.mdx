# find_and_update

Используется для поиска одной записи, применения атомарных обновлений в базе данных и возврата обновленного элемента.

---

## Что вы можете сделать

- `.cond(...)`
- `.update(...)`
- `.execute(&client)`

## Описание методов

- `.cond(...)`: определяет, какая запись будет найдена.
- `.update(...)`: применяет атомарную операцию к полю модели.
- `.execute(&client)`: выполняет обновление и возвращает обновленную запись.

## Возвращаемое значение

Возвращаемое значение `find_and_update`:

```rust
DinocoResult<M>
```

## Базовый пример

```rust
let task = dinoco::find_and_update::<Task>()
    .cond(|x| x.id.eq(task_id.clone()))
    .update(|x| x.status.set(TaskStatus::REVIEW))
    .execute(&client)
    .await?;
```

## Доступные операции в `ModelUpdate`

- `set(value)`
- `increment(value)`
- `decrement(value)`
- `multiply(value)`
- `division(value)`

## Примечания

- Обновление выполняется в одном `UPDATE` запросе.
- Если ни одна строка не соответствует условию, будет возвращена ошибка `DinocoError::RecordNotFound`.
- DSL обновления не раскрывает отношения.
- В настоящее время поток поддерживает простой первичный ключ для поиска и возврата обновленной записи.
