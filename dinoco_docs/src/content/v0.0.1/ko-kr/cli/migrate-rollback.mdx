# 사용 방법

명령어 `dinoco migrate rollback`은(는) 마지막으로 적용된 마이그레이션을 되돌립니다.

이는 데이터베이스의 마지막 변경 사항을 되돌려야 할 때 유용합니다.

---

## 명령이 하는 일

이 명령을 실행하면 CLI는 다음을 시도합니다:

- 마지막으로 적용된 마이그레이션을 식별합니다.
- 해당 마이그레이션을 되돌립니다.
- 데이터베이스의 마이그레이션 기록을 업데이트합니다.

## 매개변수

[NOME]: 마이그레이션 이름 (특정 마이그레이션으로 되돌리기 위해) (선택 사항)

## 사용 예제

```bash
dinoco migrate rollback
```

## 사용 시점

다음과 같은 경우 이 명령을 사용하십시오:

- 마지막 마이그레이션을 되돌려야 할 때
- 스키마에서 최근 문제를 발견했을 때
- 데이터베이스 개발의 개발 흐름을 조정할 때

## 주의사항

- 모든 구조적 변경이 사소하며 영향 없이 되돌릴 수 있는 것은 아닙니다.
- 중요한 환경에서 롤백을 수행하기 전에 마이그레이션의 영향을 확인하십시오.

## 다음 단계

스키마를 조정한 후 새 마이그레이션을 생성할 수 있습니다:

```bash
dinoco migrate generate
```
