# insert_many

배치 삽입에 사용됩니다.

---

## 할 수 있는 것

- `.values(Vec<M>)`: 배치로 삽입할 레코드를 정의합니다.
- `.with_relation(Vec<R>)`: 각 부모 요소에 대해 정확히 하나의 새로운 관련 엔티티를 삽입합니다.
- `.with_relations(Vec<Vec<R>>)`: 각 부모 요소에 대해 여러 개의 새로운 관련 엔티티를 삽입합니다.
- `.with_connection(Vec<R>)`: 삽입된 각 부모 요소에 대해 정확히 하나의 기존 관계를 연결합니다.
- `.with_connections(Vec<Vec<R>>)`: 삽입된 각 부모 요소에 대해 여러 개의 기존 관계를 연결합니다. 일반적으로 다대다 시나리오에서 사용됩니다.
- `.returning::<T>()`: 반환 값을 삽입된 요소의 타입 지정된 목록으로 변경합니다.
- `.execute(&client)`: 배치 작업을 실행합니다.

## 반환값

`.returning::<T>()`가 없으면 반환 값은 다음과 같습니다:

```rust
DinocoResult<()>
```

`.returning::<T>()`가 있으면 반환 값은 다음과 같습니다:

```rust
DinocoResult<Vec<T>>
```

## 기본 예제

```rust
dinoco::insert_many::<User>()
    .values(vec![
        User { id: "u1".into(), email: "a@acme.com".into(), name: "A".into() },
        User { id: "u2".into(), email: "b@acme.com".into(), name: "B".into() },
    ])
    .execute(&client)
    .await?;
```

## 반환값 예제

```rust
let created = dinoco::insert_many::<User>()
    .values(vec![
        User { id: 2, name: "Ana".to_string() },
        User { id: 3, name: "Caio".to_string() },
    ])
    .returning::<User>()
    .execute(&client)
    .await?;
```

## `with_relation(...)` 예제

각 부모 요소가 생성되고 연결되어야 할 때 `with_relation(...)`을 사용합니다.

```rust
dinoco::insert_many::<Post>()
    .values(vec![
        Team { id: "team-11".into(), name: "Data".into() },
        Team { id: "team-12".into(), name: "DX".into() },
    ])
    .with_relation(vec![
        Member { id: "member-11".into(), name: "Rafa".into(), teamId: "legacy".into() },
        Member { id: "member-12".into(), name: "Bia".into(), teamId: "legacy".into() },
    ])
    .execute(&client)
    .await?;
```

## `with_relations(...)` 예제

각 부모 레코드가 하나 이상의 자식을 가질 때 `with_relations(...)`을 사용합니다. 레코드는 자동으로 생성되고 연결됩니다.

```rust
dinoco::insert_many::<Post>()
    .values(vec![
        Team { id: "team-11".into(), name: "Data".into() },
        Team { id: "team-12".into(), name: "DX".into() },
    ])
    .with_relation(vec![
        vec![
			Member { id: "member-11".into(), name: "Rafa".into(), teamId: "legacy".into() },
        	Member { id: "member-12".into(), name: "Bia".into(), teamId: "legacy".into() },
		],

		vec![Member { id: "member-11".into(), name: "Rafa".into(), teamId: "legacy".into() }]
    ])
    .execute(&client)
    .await?;
```

## `with_connection(...)` 예제

각 부모 요소가 정확히 하나의 기존 레코드와 연결되어야 할 때 `with_connection(...)`을 사용합니다.

```rust
dinoco::insert_many::<Team>()
    .values(vec![
        Team { id: "team-11".into(), name: "Data".into() },
        Team { id: "team-12".into(), name: "DX".into() },
    ])
    .with_connection(vec![
        Member { id: "member-11".into(), name: "Rafa".into(), teamId: "legacy".into() },
        Member { id: "member-12".into(), name: "Bia".into(), teamId: "legacy".into() },
    ])
    .execute(&client)
    .await?;
```

## `with_connections(...)` 예제

각 부모 요소가 여러 개의 기존 레코드와 연결되어야 할 때 `with_connections(...)`을 사용합니다.
이는 다대다 관계에서 흔히 사용됩니다.

```rust
dinoco::insert_many::<Article>()
    .values(vec![
        Article { id: "article-11".into(), title: "Connect Multiple".into() },
        Article { id: "article-12".into(), title: "Connect Batch".into() },
    ])
    .with_connections(vec![
        vec![
            Label { id: "label-11".into(), name: "orm".into() },
            Label { id: "label-12".into(), name: "rust".into() },
        ],
        vec![
            Label { id: "label-10".into(), name: "backend".into() },
        ],
    ])
    .execute(&client)
    .await?;
```

## 참고 사항

- ``with_relation(...)` 및 `with_relations(...)`는 부모 요소가 메모리에 이미 알려진 `id`를 가지고 있을 때 가장 잘 작동합니다.`
- ``with_connection(...)` 및 `with_connections(...)`는 기존 관계를 연결하기 위한 것입니다.`
- ``with_connection(...)`은 `values(...)`와 동일한 수의 요소를 필요로 합니다.`
- ``with_connections(...)`는 주 벡터에 비례하여 동일한 수의 그룹을 필요로 합니다.`
