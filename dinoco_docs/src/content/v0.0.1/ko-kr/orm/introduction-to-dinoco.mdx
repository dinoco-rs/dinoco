# Dinoco 소개

이 라이브러리는 데이터베이스 관리를 단순화하기 위해 개발되었습니다. [Prisma](https://www.prisma.io/)의 철학에서 영감을 받은 Dinoco는 [Rust](https://www.rust-lang.org/) 생태계에 유연하고, 타입이 지정되며, 매우 생산적인 인터페이스를 제공합니다.

---

## Rust에서의 Prisma 경험

Dinoco는 Prisma의 민첩성과 강력한 타입 지정을 [Rust](https://www.rust-lang.org/) 생태계에 가져오고자 하는 열망에서 탄생했습니다. 우리는 일상적인 작업을 위해 복잡한 매크로를 구성하거나 수동 SQL을 작성하는 데 시간을 낭비해서는 안 된다고 믿습니다.

## 프로젝트의 핵심 원칙

- **DX (개발자 경험) 우선:** 인지 부하를 줄이고 개발 속도를 높이는 유연하고 직관적인 API.
- **종단 간 타입 안전성:** 데이터가 항상 타입 정의와 동기화되고 컴파일 시점에 오류가 포착되도록 보장합니다.
- **간단한 관계:** 번거로운 수동 조인을 잊으세요. 관계를 정의하면 Dinoco가 백그라운드에서 복잡성을 해결합니다.
- **강력한 CLI:** 워크플로우를 체계적이고 빠르게 유지하기 위한 통합 마이그레이션 관리 및 코드 생성.

## 우리는 어디에 위치하는가?

[Rust](https://www.rust-lang.org/) 생태계에는 강력한 도구들이 있지만, 종종 가파른 학습 곡선을 요구합니다. **Dinoco**는 현재 옵션들 사이의 간극을 메웁니다:

- **[Diesel](https://diesel.rs/)과 비교:** Diesel은 깊이 있는 SQL 지식과 복잡한 DSL(도메인 특정 언어)을 요구하는 반면, Dinoco는 간단한 쿼리를 위해 컴파일러와 씨름할 필요 없이 깔끔하고 생산성 지향적인 구문을 우선시합니다.
- **[SeaORM](https://www.sea-ql.org/SeaORM/)과 비교:** SeaORM은 훌륭하지만, 계층 기반 아키텍처는 많은 프로젝트에서 과도하게 중복될 수 있습니다. Dinoco는 불필요한 "보일러플레이트" 코드를 제거하고 훨씬 적은 코드 라인으로 동일한 견고성을 제공합니다.

[Rust](https://www.rust-lang.org/)의 성능과 Prisma가 Node.js 세계에 가져온 유연성을 찾고 있다면, Dinoco는 다음 프로젝트를 위한 올바른 선택입니다.
