export const title = "update_many";

# update_many

Utilizzato per aggiornare più record contemporaneamente.

---

## Cosa puoi fare

- `.cond(...)`: limita quali record possono essere aggiornati.
- `.values(Vec<M>)`: definisce gli elementi utilizzati nell'aggiornamento in batch.
- `.returning::<T>()`: restituisce i record aggiornati come una lista tipizzata.
- `.execute(&client)`: esegue l'aggiornamento in batch.

## Ritorno

Senza `.returning::<T>()`, il ritorno è:

```rust
DinocoResult<()>
```

Con `.returning::<T>()`, il ritorno diventa:

```rust
DinocoResult<Vec<T>>
```

## Esempio base

```rust
dinoco::update_many::<User>()
    .values(vec![
        User { id: 1, email: "a@acme.com".into(), name: "Ana".into() },
        User { id: 2, email: "b@acme.com".into(), name: "Bia".into() },
    ])
    .execute(&client)
    .await?;
```

## Esempio con ritorno

```rust
let updated = dinoco::update_many::<User>()
    .values(vec![
        User { id: 2, name: "Ana Batch".to_string() },
        User { id: 3, name: "Caio Batch".to_string() },
    ])
    .returning::<User>()
    .execute(&client)
    .await?;
```

## Esempio con filtro

```rust
dinoco::update_many::<User>()
    .cond(|x| x.active.eq(true))
    .values(vec![
        User { id: 10, email: "a@acme.com".into(), name: "Ana".into() },
        User { id: 11, email: "b@acme.com".into(), name: "Bia".into() },
    ])
    .execute(&client)
    .await?;
```
