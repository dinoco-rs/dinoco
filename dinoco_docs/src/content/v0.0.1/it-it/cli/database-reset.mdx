export const title = "Ripristino del database";

# Come si usa?

Il comando `dinoco database reset` ripristina il database configurato.

Questo è utile in fase di sviluppo quando si desidera pulire completamente lo stato attuale del database.

---

## Cosa fa il comando

Quando si esegue questo comando, la CLI tenta di rimuovere gli oggetti dal database corrente e di ricreare uno stato pulito per l'ambiente configurato.

Questo flusso è particolarmente utile per:

- Ripristinare un ambiente locale da zero
- Pulire i dati di test
- Correggere ambienti inconsistenti durante lo sviluppo

## Precauzioni

- Questo comando è distruttivo per il database configurato.
- Usare con cautela al di fuori di un ambiente locale.
- Confermare sempre che l'URL di connessione punti al database corretto.

## Passi successivi

Dopo il ripristino, di solito si esegue quanto segue:

```bash
dinoco migrate run
```

o:

```bash
dinoco migrate generate --apply
```
