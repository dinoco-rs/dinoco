# Come si usa?

Il comando `dinoco migrate run` esegue tutte le migrazioni in sospeso.

Viene utilizzato per allineare il database configurato con la cronologia delle migrazioni già generata.

---

## Cosa fa il comando

Quando si esegue questo comando, la CLI:

- Legge la cronologia delle migrazioni
- Controlla quali non sono ancora state applicate
- Esegue le migrazioni in sospeso nell'ordine corretto

## Quando usarlo

Utilizzare questo comando quando:

- Hai già generato delle migrazioni
- Vuoi applicare modifiche in sospeso al database
- Stai preparando un ambiente locale o di deployment

## Passi successivi

Se necessario, genera i modelli con:

```bash
dinoco models generate
```
