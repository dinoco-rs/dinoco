# Wie verwende ich es?

Der Befehl `dinoco database reset` setzt die konfigurierte Datenbank zurück.

Dies ist nützlich in der Entwicklung, wenn Sie den aktuellen Zustand der Datenbank vollständig löschen möchten.

---

## Was der Befehl tut

Wenn dieser Befehl ausgeführt wird, versucht die CLI, Objekte aus der aktuellen Datenbank zu entfernen und einen sauberen Zustand für die konfigurierte Umgebung neu zu erstellen.

Dieser Workflow ist besonders nützlich für:

- Eine lokale Umgebung von Grund auf neu starten
- Testdaten bereinigen
- Inkonsistente Umgebungen während der Entwicklung korrigieren

## Vorsichtsmaßnahmen

- Dieser Befehl ist zerstörerisch für die konfigurierte Datenbank.
- Außerhalb einer lokalen Umgebung mit Vorsicht verwenden.
- Stellen Sie immer sicher, dass die Verbindungs-URL auf die richtige Datenbank verweist.

## Nächste Schritte

Nach dem Zurücksetzen führen Sie normalerweise aus:

```bash
dinoco migrate run
```

oder:

```bash
dinoco migrate generate --apply
```
