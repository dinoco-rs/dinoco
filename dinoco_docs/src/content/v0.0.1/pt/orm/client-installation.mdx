# Instalacao do client

O Dinoco Client e a porta de entrada para consultar dados da aplicacao a partir do schema gerado.

## Dependencias

Instale o pacote principal no projeto e mantenha o CLI disponivel para gerar artefatos quando o schema mudar.

```bash
npm install dinoco
```

## Generate

Depois de alterar o schema, gere novamente o client para refletir os novos modelos, enums e relacoes.

```bash
dinoco generate
```

## Conexao

Centralize a criacao do client em um modulo unico para reaproveitar a conexao entre handlers, jobs e scripts.

```ts
import { DinocoClient } from 'dinoco';

export const db = new DinocoClient({
	url: process.env.DATABASE_URL,
});
```

## Validacao

Antes de subir a aplicacao, valide se a conexao esta correta, se o schema gerou sem avisos e se o ambiente expoe as variaveis esperadas.
