# Modelo

O schema concentra a definicao de modelos, relacoes e geracao de tipos da aplicacao.

## Campos

- Use tipos primitivos para dados basicos.
- Marque chaves e unicidade no proprio modelo.
- Documente defaults no schema para manter clareza.

## Relacoes

```prisma
model User {
  id    Int    @id @default(autoincrement())
  email String @unique
  posts Post[]
}
```

## Schema Dinoco

```dinoco
config {
  provider = "postgres"
}

model User {
  id Integer @id @default(autoincrement())
  email String @unique
  profile Profile?

  @@table_name("users")
}

model Profile {
  id Integer @id
  bio String?
}
```

## Generate

Depois de ajustar o schema, rode o generate para atualizar os tipos e o client.
