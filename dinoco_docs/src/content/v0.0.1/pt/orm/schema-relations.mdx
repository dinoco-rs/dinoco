# Relacoes

Relacoes tornam o schema mais expressivo e ajudam o client a inferir joins, includes e seguranca de tipos.

## One-to-one

Use relacoes um-para-um quando uma entidade possuir uma extensao unica, como perfil, configuracao ou metadata privada.

## One-to-many

Esse formato cobre os casos mais comuns, como usuario e posts, organizacao e membros ou pedido e itens.

```dinoco
model User {
	id    String @id
	posts Post[]
}

model Post {
	id      String @id
	userId  String
	user    User   @relation(fields: [userId], references: [id])
}
```

## Muitos-para-muitos

Para relacoes complexas, use uma tabela intermediaria quando precisar guardar contexto adicional entre as duas entidades.

## Boas praticas

Mantenha nomes consistentes, escolha chaves claras e documente relacoes que tenham impacto operacional importante.
