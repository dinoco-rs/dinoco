# Migrations

Migrations ajudam a evoluir o schema sem perder rastreabilidade entre desenvolvimento, review e producao.

## Create

Crie uma migration sempre que uma mudanca estrutural precisar ser aplicada de forma reproduzivel.

```bash
dinoco migrate dev --name add-user-profile
```

## Apply

Em ambientes controlados, aplique migrations pendentes como parte do deploy ou de um passo dedicado do pipeline.

## Rollback

Quando necessario, planeje rollback com antecedencia e saiba quais mudancas sao reversiveis sem perda de dados.

## Ambientes

Mantenha a mesma disciplina entre local, staging e producao para evitar drift entre o schema esperado e o banco real.
