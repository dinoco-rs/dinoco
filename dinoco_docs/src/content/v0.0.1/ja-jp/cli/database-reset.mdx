# 使い方

`dinoco database reset` コマンドは、設定されたデータベースをリセットします。

開発中にデータベースの現在の状態を完全にクリーンアップしたいときに便利です。

---

## コマンドの動作

このコマンドを実行すると、CLI は現在のデータベースからオブジェクトを削除し、設定された環境向けにクリーンな状態を再作成しようとします。

このプロセスは特に以下の場合に役立ちます:

- ローカル環境を最初から再起動する
- テストデータをクリーンアップする
- 開発中に一貫性のない環境を修正する

## 注意点

- このコマンドは設定されたデータベースに対して破壊的です。
- ローカル環境以外での使用は注意してください。
- 接続 URL が正しいデータベースを指していることを必ず確認してください。

## 次のステップ

リセット後、通常は以下を実行します:

```bash
dinoco migrate run
```

または:

```bash
dinoco migrate generate --apply
```
