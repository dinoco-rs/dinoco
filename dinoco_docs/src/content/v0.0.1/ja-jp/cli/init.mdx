# 使用方法

`dinoco init` コマンドは、現在のプロジェクトでDinoco環境を初期化します。

スキーマ、マイグレーション、モデル生成の作業を開始するために必要な基盤を準備します。

---

## コマンドの機能

`dinoco init` を実行すると、CLIはDinoco環境の初期設定をガイドします。

一般的に、このフローは以下を準備します。

- `dinoco` ディレクトリの基本構造
- 初期スキーマファイル
- データベース設定
- 将来のマイグレーションの基盤

## 使用するタイミング

このコマンドは、以下の状況で使用します。

- プロジェクトでDinocoを初めて設定する場合
- ローカル環境の初期構造を作成したい場合
- マイグレーションとモデルを生成するためにプロジェクトを準備する必要がある場合

## 次のステップ

`init` の後、最も一般的なフローは次のとおりです。

```bash
dinoco migrate generate
```

または、すぐに生成して適用したい場合は、次のとおりです。

```bash
dinoco migrate generate --apply
```
