# 使い方

`dinoco init` コマンドは、現在のプロジェクトで Dinoco 環境を初期化します。

スキーマ、マイグレーション、モデル生成を開始するために必要な基盤を準備します。

---

## コマンドの動作

`dinoco init` を実行すると、CLI が Dinoco 環境の初期設定を案内します。

一般的に、この流れでは次を準備します:

- `dinoco` ディレクトリの基本構造
- 初期 schema ファイル
- データベース設定
- 今後の migration のための土台

## 使うタイミング

次のようなときにこのコマンドを使います:

- プロジェクトで初めて Dinoco をセットアップするとき
- ローカル環境の初期構造を作りたいとき
- migration とモデル生成の準備をしたいとき

## 次のステップ

`init` の後で最も一般的な流れは次のとおりです:

```bash
dinoco migrate generate
```

または、すぐに生成して適用したい場合は:

```bash
dinoco migrate generate --apply
```
