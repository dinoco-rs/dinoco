export const title = "update";

# update

フィルタリングされたレコードを更新するために使用されます。

---

## できること

- `.cond(...)`: 更新するレコードを定義します。
- `.values(item)`: レコードの新しい値を指定します。
- `.connect(...)`: 書き込み操作でサポートされているリレーションシップリンクを作成します。
- `.disconnect(...)`: 書き込み操作でサポートされているリレーションシップリンクを削除します。
- `.returning::<T>()`: 更新されたレコードを型付きプロジェクションで返します。
- `.execute(&client)`: データベースで更新を実行します。

## 戻り値

`.returning::<T>()` がない場合の戻り値は次のとおりです。

```rust
DinocoResult<()>
```

`.returning::<T>()` がある場合の戻り値は次のとおりです。

```rust
DinocoResult<Vec<T>>
```

注:

- `update().returning()` は、`.connect(...)` または `.disconnect(...)` を使用したリレーショナルな書き込み操作をサポートしていません。

## フィールド更新の例

```rust
dinoco::update::<User>()
    .cond(|w| w.id.eq(10))
    .values(User {
        id: 10,
        email: "novo@acme.com".to_string(),
        name: "Novo Nome".to_string(),
    })
    .execute(&client)
    .await?;
```

## `connect(...)` の例

書き込み操作でサポートされているリレーションシップ（通常は多対多）を接続するために使用されます。

```rust
dinoco::update::<User>()
    .cond(|w| w.id.eq(10))
    .connect(|r| r.roles().slug.eq("admin"))
    .execute(&client)
    .await?;
```

## `disconnect(...)` の例

リレーションシップを切断するために使用されます。

```rust
dinoco::update::<User>()
    .cond(|w| w.id.eq(10))
    .disconnect(|r| r.roles().slug.eq("guest"))
    .execute(&client)
    .await?;
```

## connect と disconnect で使えるフィルター

- `eq`
- `neq`
- `gt`
- `gte`
- `lt`
- `lte`
- `in_values`
- `not_in_values`
- `is_null`
- `is_not_null`
- `includes`
- `starts_with`
- `ends_with`
