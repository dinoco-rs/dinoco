export const title = "insert_many";

# insert_many

バッチ挿入に使用されます。

---

## できること

- `.values(Vec\<M\>)`: バッチで挿入されるレコードを定義します。
- `.with_relation(Vec\<R\>)`: 各親に対して、正確に1つの新しい関連レコードを挿入します。
- `.with_relations(Vec\<Vec\<R\>\>)` : 各親に対して、複数の新しい関連レコードを挿入します。
- `.with_connection(Vec\<R\>)`: 挿入された各親に対して、正確に1つの既存の関連を接続します。
- `.with_connections(Vec\<Vec\<R\>\>)` : 挿入された各親に対して、複数の既存の関連を接続します。これは通常、多対多のシナリオで使用されます。
- `.returning::\<T\>()`: 戻り値を、挿入されたアイテムの型付きリストに変更します。
- `.execute(&client)`: バッチ操作を実行します。

## 戻り値

`.returning::\<T\>()`がない場合、戻り値は次のようになります。

```rust
DinocoResult<()>
```

`.returning::\<T\>()`がある場合、戻り値は次のようになります。

```rust
DinocoResult<Vec<T>>
```

## シンプルな例

```rust
dinoco::insert_many::<User>()
    .values(vec![
        User { id: "u1".into(), email: "a@acme.com".into(), name: "A".into() },
        User { id: "u2".into(), email: "b@acme.com".into(), name: "B".into() },
    ])
    .execute(&client)
    .await?;
```

## 戻り値を含む例

```rust
let created = dinoco::insert_many::<User>()
    .values(vec![
        User { id: 2, name: "Ana".to_string() },
        User { id: 3, name: "Caio".to_string() },
    ])
    .returning::<User>()
    .execute(&client)
    .await?;
```

## with_relation(...) の例

各親が作成され、接続される必要がある場合に `with_relation(...)` を使用します。

```rust
dinoco::insert_many::<Post>()
    .values(vec![
        Team { id: "team-11".into(), name: "Data".into() },
        Team { id: "team-12".into(), name: "DX".into() },
    ])
    .with_relation(vec![
        Member { id: "member-11".into(), name: "Rafa".into(), teamId: "legacy".into() },
        Member { id: "member-12".into(), name: "Bia".into(), teamId: "legacy".into() },
    ])
    .execute(&client)
    .await?;
```

## with_relations(...) の例

各親レコードが複数の子を持つ場合に `with_relations(...)` を使用します。レコードは自動的に作成され、リンクされます。

```rust
dinoco::insert_many::<Post>()
    .values(vec![
        Team { id: "team-11".into(), name: "Data".into() },
        Team { id: "team-12".into(), name: "DX".into() },
    ])
    .with_relation(vec![
        vec![
			Member { id: "member-11".into(), name: "Rafa".into(), teamId: "legacy".into() },
        	Member { id: "member-12".into(), name: "Bia".into(), teamId: "legacy".into() },
		],

		vec![Member { id: "member-11".into(), name: "Rafa".into(), teamId: "legacy".into() }]
    ])
    .execute(&client)
    .await?;
```

## with_connection(...) の例

各親が正確に1つの既存のレコードに接続される必要がある場合に `with_connection(...)` を使用します。

```rust
dinoco::insert_many::<Team>()
    .values(vec![
        Team { id: "team-11".into(), name: "Data".into() },
        Team { id: "team-12".into(), name: "DX".into() },
    ])
    .with_connection(vec![
        Member { id: "member-11".into(), name: "Rafa".into(), teamId: "legacy".into() },
        Member { id: "member-12".into(), name: "Bia".into(), teamId: "legacy".into() },
    ])
    .execute(&client)
    .await?;
```

## with_connections(...) の例

各親が複数の既存のレコードに接続される必要がある場合に `with_connections(...)` を使用します。
これは多対多のリレーションシップで一般的な使用法です。

```rust
dinoco::insert_many::<Article>()
    .values(vec![
        Article { id: "article-11".into(), title: "Connect Multiple".into() },
        Article { id: "article-12".into(), title: "Connect Batch".into() },
    ])
    .with_connections(vec![
        vec![
            Label { id: "label-11".into(), name: "orm".into() },
            Label { id: "label-12".into(), name: "rust".into() },
        ],
        vec![
            Label { id: "label-10".into(), name: "backend".into() },
        ],
    ])
    .execute(&client)
    .await?;
```

## 備考

- `with_relation(...)` と `with_relations(...)` は、親がメモリ内で既知のIDを持っている場合に最も効果的です。
- `with_connection(...)` と `with_connections(...)` は、既存のリレーションシップを接続することを目的としています。
- `with_connection(...)` は `values(...)` と同じ数のアイテムを必要とします。
- `with_connections(...)` は、メインベクターに対して同じ数のグループを必要とします。
