# 如何使用？

命令 `dinoco migrate rollback` 会回滚最后一次应用的迁移。

当您需要撤销数据库中最近的更改时，这很有用。

---

## 命令的作用

执行此命令时，CLI 会尝试：

- 识别最后一次应用的迁移
- 回滚该迁移
- 更新数据库的迁移历史记录

## 参数

[NOME]: 迁移名称（用于回滚到特定迁移）（可选）

## 使用示例

```bash
dinoco migrate rollback
```

## 何时使用

在以下情况下使用此命令：

- 最后一次迁移需要回滚
- 您在架构中发现了最近的问题
- 您正在开发环境中调整数据库演进流程

## 注意事项

- 并非所有结构性更改都可以轻松无影响地回滚。
- 在重要环境中执行回滚之前，请仔细审查迁移的影响。

## 后续步骤

调整架构后，您可以生成新的迁移：

```bash
dinoco migrate generate
```
