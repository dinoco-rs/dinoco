# 枚举

枚举允许将 Dinoco 模式中的字段限制为一组固定的已知值。

当值可预测、经过验证且需要在多个模型中重用时，它们非常有用。

---

## 什么是枚举

`enum` 定义了一个可能的封闭值列表。

```dinoco
enum Role {
	USER
	ADMIN
}
```

在这种情况下，`Role` 只能是 `USER` 或 `ADMIN`。

## 在模型中的使用

定义后，枚举可以在任何模型中用作字段类型。

```dinoco
enum Role {
	USER
	ADMIN
}

model User {
	id   Integer @id @default(autoincrement())
	role Role    @default(USER)
}
```

这里：

- `role` 使用了 `Role` 枚举。
- `@default(USER)` 定义了字段的默认值。

## 何时使用枚举

枚举对于表示以下值很有用：

- 用户角色
- 发布状态
- 工作流步骤
- 支付情况

示例：

```dinoco
enum PostStatus {
	DRAFT
	REVIEW
	PUBLISHED
	ARCHIVED
}

model Post {
	id     Integer    @id @default(autoincrement())
	title  String
	status PostStatus @default(DRAFT)
}
```

## 最佳实践

- 当可能的值已知且有限时，使用枚举。
- 枚举名称首选 PascalCase，值首选 UPPER_CASE。
- 当存在自然的初始状态时，使用 `@default(...)`。
