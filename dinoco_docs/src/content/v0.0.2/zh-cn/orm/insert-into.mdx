export const title = 'insert_into';

# insert_into

用于插入一条记录。

---

## 您可以做什么

- `.values(item)`: 定义要插入的记录。
- `.returning::<T>()`: 将返回值更改为插入项的类型化投影。
- `.execute(&client)`: 执行数据库写入操作。

## 使用 Extend #[insertable] 的示例

当您希望 `.values(...)` 接受包含新关系和现有连接的更丰富的有效载荷时，请将 `#[insertable]` 与 `#[derive(Extend)]` 一起使用。Dinoco 会首先插入父级，然后递归处理所有内容。

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(User)]
#[insertable]
struct UserWithPosts {
    id: i64,
    name: String,
    posts: Vec<PostWithComments>,
}

#[derive(Debug, Clone, dinoco::Extend)]
#[extend(Post)]
#[insertable]
struct PostWithComments {
    id: i64,
    title: String,
    published: bool,
    authorId: i64,
    comments: Vec<Comment>,
}

dinoco::insert_into::<User>()
    .values(UserWithPosts {
        id: 1,
        name: "安娜".to_string(),
        posts: vec![
            PostWithComments {
                id: 10,
                title: "递归插入".to_string(),
                published: true,
                authorId: 0,
                comments: vec![Comment { id: 100, text: "第一个".to_string(), flagged: false, postId: 0 }],
            },
        ],
    })
    .execute(&client)
    .await?;
```

## 嵌套关系的示例

使用带有 `#[insertable]` 的 `Extend` 有效载荷来创建父级，并在同一流程中创建相关项。

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(User)]
#[insertable]
struct UserWithProfile {
    id: String,
    username: String,
    role: Role,
    profile: Option<Profile>,
}

dinoco::insert_into::<User>()
    .values(UserWithProfile {
        id: "user-1".to_string(),
        username: "比亚".to_string(),
        role: Role::USER,
        profile: Some(Profile {
            id: "profile-1".to_string(),
            bio: Some("软件工程师".to_string()),
            userId: String::new(),
        }),
    })
    .execute(&client)
    .await?;
```

## 嵌套连接的示例

当关系指向已存在的记录时，代码生成器会生成一个像 `ArticleConnection` 这样的枚举，用于有效载荷中。

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(Article)]
#[insertable]
struct ArticleWithLabels {
    id: String,
    title: String,
    labels: Vec<ArticleConnection>,
}

dinoco::insert_into::<Article>()
    .values(ArticleWithLabels {
        id: "article-10".into(),
        title: "连接现有".into(),
        labels: vec![ArticleConnection::Label("label-10".into())],
    })
    .execute(&client)
    .await?;
```

## ArticleConnection 示例

这是在多对多关系中连接现有记录的最常见格式。

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(Article)]
#[insertable]
struct ArticleWithLabels {
    id: String,
    title: String,
    labels: Vec<ArticleConnection>,
}

let value = ArticleWithLabels {
    id: "article-20".into(),
    title: "连接现有".into(),
    labels: vec![
        ArticleConnection::Label("label-10".into()),
        ArticleConnection::Label("label-11".into()),
    ],
};

dinoco::insert_into::<Article>()
    .values(value)
    .execute(&client)
    .await?;
```

## 返回值

如果没有 `.returning::<T>()`，返回值是：

```rust
DinocoResult<()>
```

有了 `.returning::<T>()`，返回值变为：

```rust
DinocoResult<T>
```

## 简单示例

```rust
dinoco::insert_into::<User>()
    .values(User {
        id: "usr_1".to_string(),
        email: "ana@acme.com".to_string(),
        name: "安娜".to_string(),
    })
    .execute(&client)
    .await?;
```

## 带类型返回值的示例

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(User)]
struct UserSummary {
    id: i64,
    name: String,
}

let created = dinoco::insert_into::<User>()
    .values(User { id: 1, name: "马修斯".to_string() })
    .returning::<UserSummary>()
    .execute(&client)
    .await?;
```
