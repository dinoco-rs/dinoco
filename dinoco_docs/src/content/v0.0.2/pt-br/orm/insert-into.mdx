export const title = "insert_into";

# insert_into

Usado para inserir um registro.

---

## O que você pode fazer

- Passar valores com `.values(item)`
- Inserir um registro simples
- Inserir um registro com relações novas aninhadas
- Inserir um registro com conexões para itens já existentes
- Executar com `.execute(&client)`

## Descrição dos métodos

- `.values(item)`: define o registro que será inserido.
- `#[derive(Extend)]`: permite criar payloads maiores que o model base.
- `#[insertable]`: faz o Dinoco inserir o pai e processar relações e conexões aninhadas dentro de `.values(...)`.
- `.returning::<T>()`: muda o retorno para uma projeção tipada do item inserido.
- `.execute(&client)`: executa a escrita no banco.

## Exemplo com Extend #[insertable]

Use `#[insertable]` junto com `#[derive(Extend)]` quando você quiser que `.values(...)` aceite um payload mais rico com relações novas e conexões existentes. O Dinoco insere o pai primeiro e depois processa tudo de forma recursiva.

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(User)]
#[insertable]
struct UserWithPosts {
    id: i64,
    name: String,
    posts: Vec<PostWithComments>,
}

#[derive(Debug, Clone, dinoco::Extend)]
#[extend(Post)]
#[insertable]
struct PostWithComments {
    id: i64,
    title: String,
    published: bool,
    authorId: i64,
    comments: Vec<Comment>,
}

dinoco::insert_into::<User>()
    .values(UserWithPosts {
        id: 1,
        name: "Ana".to_string(),
        posts: vec![
            PostWithComments {
                id: 10,
                title: "Insert recursivo".to_string(),
                published: true,
                authorId: 0,
                comments: vec![Comment { id: 100, text: "primeiro".to_string(), flagged: false, postId: 0 }],
            },
        ],
    })
    .execute(&client)
    .await?;
```

## Exemplo com relação aninhada

Use um payload `Extend` com `#[insertable]` para criar o pai e também criar o relacionado no mesmo fluxo.

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(User)]
#[insertable]
struct UserWithProfile {
    id: String,
    username: String,
    role: Role,
    profile: Option<Profile>,
}

dinoco::insert_into::<User>()
    .values(UserWithProfile {
        id: "user-1".to_string(),
        username: "bia".to_string(),
        role: Role::USER,
        profile: Some(Profile {
            id: "profile-1".to_string(),
            bio: Some("Engenheira de software".to_string()),
            userId: String::new(),
        }),
    })
    .execute(&client)
    .await?;
```

## Exemplo com conexão aninhada

Quando a relação aponta para registros já existentes, o codegen gera um enum como `ArticleConnection` para ser usado dentro do payload.

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(Article)]
#[insertable]
struct ArticleWithLabels {
    id: String,
    title: String,
    labels: Vec<ArticleConnection>,
}

dinoco::insert_into::<Article>()
    .values(ArticleWithLabels {
        id: "article-10".into(),
        title: "Connect Existing".into(),
        labels: vec![ArticleConnection::Label("label-10".into())],
    })
    .execute(&client)
    .await?;
```

## Exemplo com ArticleConnection

Esse é o formato mais comum para conexão de registros existentes em relações many-to-many.

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(Article)]
#[insertable]
struct ArticleWithLabels {
    id: String,
    title: String,
    labels: Vec<ArticleConnection>,
}

let value = ArticleWithLabels {
    id: "article-20".into(),
    title: "Connect Existing".into(),
    labels: vec![
        ArticleConnection::Label("label-10".into()),
        ArticleConnection::Label("label-11".into()),
    ],
};

dinoco::insert_into::<Article>()
    .values(value)
    .execute(&client)
    .await?;
```

## Retorno

Sem `.returning::<T>()`, o retorno é:

```rust
DinocoResult<()>
```

Com `.returning::<T>()`, o retorno passa a ser:

```rust
DinocoResult<T>
```

## Exemplo simples

```rust
dinoco::insert_into::<User>()
    .values(User {
        id: "usr_1".to_string(),
        email: "ana@acme.com".to_string(),
        name: "Ana".to_string(),
    })
    .execute(&client)
    .await?;
```

## Exemplo com retorno tipado

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(User)]
struct UserSummary {
    id: i64,
    name: String,
}

let created = dinoco::insert_into::<User>()
    .values(User { id: 1, name: "Matheus".to_string() })
    .returning::<UserSummary>()
    .execute(&client)
    .await?;
```
