export const title = "insert_many";

# insert_many

Wird für die Batch-Einfügung verwendet.

---

## Was Sie tun können

- `.values(Vec<M>)`: Definiert die Datensätze, die im Batch eingefügt werden sollen.
- `.with_relation(Vec<R>)`: Fügt genau einen neuen verknüpften Datensatz für jedes Elternteil ein.
- `.with_relations(Vec<Vec<R>>)`: Fügt mehrere neue verknüpfte Datensätze für jedes Elternteil ein.
- `.with_connection(Vec<R>)`: Verbindet genau eine bereits bestehende Beziehung für jedes eingefügte Elternteil.
- `.with_connections(Vec<Vec<R>>)`: Verbindet mehrere bereits bestehende Beziehungen für jedes eingefügte Elternteil, typischerweise in Many-to-Many-Szenarien.
- `.returning::<T>()`: Ändert den Rückgabewert zu einer typisierten Liste der eingefügten Elemente.
- `.execute(&client)`: Führt den Batch-Vorgang aus.

## Rückgabewert

Ohne `.returning::<T>()` ist der Rückgabewert:

```rust
DinocoResult<()>
```

Mit `.returning::<T>()` ändert sich der Rückgabewert zu:

```rust
DinocoResult<Vec<T>>
```

## Einfaches Beispiel

```rust
dinoco::insert_many::<User>()
    .values(vec![
        User { id: "u1".into(), email: "a@acme.com".into(), name: "A".into() },
        User { id: "u2".into(), email: "b@acme.com".into(), name: "B".into() },
    ])
    .execute(&client)
    .await?;
```

## Beispiel mit Rückgabewert

```rust
let created = dinoco::insert_many::<User>()
    .values(vec![
        User { id: 2, name: "Ana".to_string() },
        User { id: 3, name: "Caio".to_string() },
    ])
    .returning::<User>()
    .execute(&client)
    .await?;
```

## Beispiel mit with_relation(...)

Verwenden Sie `with_relation(...)`, wenn jedes Elternteil erstellt und verbunden werden muss.

```rust
dinoco::insert_many::<Post>()
    .values(vec![
        Team { id: "team-11".into(), name: "Data".into() },
        Team { id: "team-12".into(), name: "DX".into() },
    ])
    .with_relation(vec![
        Member { id: "member-11".into(), name: "Rafa".into(), teamId: "legacy".into() },
        Member { id: "member-12".into(), name: "Bia".into(), teamId: "legacy".into() },
    ])
    .execute(&client)
    .await?;
```

## Beispiel mit with_relations(...)

Verwenden Sie `with_relations(...)`, wenn jeder übergeordnete Datensatz mehr als ein Kind hat; die Datensätze werden automatisch erstellt und verknüpft.

```rust
dinoco::insert_many::<Post>()
    .values(vec![
        Team { id: "team-11".into(), name: "Data".into() },
        Team { id: "team-12".into(), name: "DX".into() },
    ])
    .with_relation(vec![
        vec![
			Member { id: "member-11".into(), name: "Rafa".into(), teamId: "legacy".into() },
        	Member { id: "member-12".into(), name: "Bia".into(), teamId: "legacy".into() },
		],

		vec![Member { id: "member-11".into(), name: "Rafa".into(), teamId: "legacy".into() }]
    ])
    .execute(&client)
    .await?;
```

## Beispiel mit with_connection(...)

Verwenden Sie `with_connection(...)`, wenn jedes Elternteil mit genau einem bereits bestehenden Datensatz verbunden werden muss.

```rust
dinoco::insert_many::<Team>()
    .values(vec![
        Team { id: "team-11".into(), name: "Data".into() },
        Team { id: "team-12".into(), name: "DX".into() },
    ])
    .with_connection(vec![
        Member { id: "member-11".into(), name: "Rafa".into(), teamId: "legacy".into() },
        Member { id: "member-12".into(), name: "Bia".into(), teamId: "legacy".into() },
    ])
    .execute(&client)
    .await?;
```

## Beispiel mit with_connections(...)

Verwenden Sie `with_connections(...)`, wenn jedes Elternteil mit mehreren bereits bestehenden Datensätzen verbunden werden muss.
Dies ist eine übliche Verwendung in Many-to-Many-Beziehungen.

```rust
dinoco::insert_many::<Article>()
    .values(vec![
        Article { id: "article-11".into(), title: "Connect Multiple".into() },
        Article { id: "article-12".into(), title: "Connect Batch".into() },
    ])
    .with_connections(vec![
        vec![
            Label { id: "label-11".into(), name: "orm".into() },
            Label { id: "label-12".into(), name: "rust".into() },
        ],
        vec![
            Label { id: "label-10".into(), name: "backend".into() },
        ],
    ])
    .execute(&client)
    .await?;
```

## Anmerkungen

- `with_relation(...)` und `with_relations(...)` funktionieren am besten, wenn das Elternteil bereits eine bekannte ID im Speicher hat.
- `with_connection(...)` und `with_connections(...)` sind für die Verbindung bereits bestehender Beziehungen gedacht.
- `with_connection(...)` erfordert die gleiche Anzahl von Elementen wie `values(...)`.
- `with_connections(...)` erfordert die gleiche Anzahl von Gruppen in Bezug auf den Hauptvektor.
