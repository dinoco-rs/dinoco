export const title = "update";

# update

Verwendet, um gefilterte Datensätze zu aktualisieren.

---

## Was Sie tun können

- `.cond(...)`: definiert, welche Datensätze aktualisiert werden sollen.
- `.values(item)`: gibt die neuen Werte des Datensatzes an.
- `.connect(...)`: erstellt unterstützte Beziehungsknüpfungen zum Schreiben.
- `.disconnect(...)`: entfernt unterstützte Beziehungsknüpfungen zum Schreiben.
- `.returning::<T>()`: gibt die aktualisierten Datensätze in einer typisierten Projektion zurück.
- `.execute(&client)`: führt die Aktualisierung in der Datenbank aus.

## Rückgabe

Ohne `.returning::<T>()`, ist die Rückgabe:

```rust
DinocoResult<()>
```

Mit `.returning::<T>()`, wird die Rückgabe zu:

```rust
DinocoResult<Vec<T>>
```

Hinweis:

- `update().returning()` unterstützt keine relation writes mit `.connect(...)` oder `.disconnect(...)`.

## Beispiel für das Aktualisieren von Feldern

```rust
dinoco::update::<User>()
    .cond(|w| w.id.eq(10))
    .values(User {
        id: 10,
        email: "novo@acme.com".to_string(),
        name: "Novo Nome".to_string(),
    })
    .execute(&client)
    .await?;
```

## Beispiel mit connect(...)

Wird verwendet, um unterstützte Beziehungen zum Schreiben zu verbinden, typischerweise Many-to-Many.

```rust
dinoco::update::<User>()
    .cond(|w| w.id.eq(10))
    .connect(|r| r.roles().slug.eq("admin"))
    .execute(&client)
    .await?;
```

## Beispiel mit disconnect(...)

Wird verwendet, um Beziehungen zu trennen.

```rust
dinoco::update::<User>()
    .cond(|w| w.id.eq(10))
    .disconnect(|r| r.roles().slug.eq("guest"))
    .execute(&client)
    .await?;
```

## Verfügbare Filter in connect und disconnect

- `eq`
- `neq`
- `gt`
- `gte`
- `lt`
- `lte`
- `in_values`
- `not_in_values`
- `is_null`
- `is_not_null`
- `includes`
- `starts_with`
- `ends_with`
