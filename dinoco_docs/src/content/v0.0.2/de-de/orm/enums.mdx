# Enums

Enums ermöglichen es, ein Feld auf eine feste Menge bekannter Werte im Dinoco-Schema zu beschränken.

Sie sind nützlich, wenn der Wert vorhersehbar, validiert und über Models hinweg wiederverwendet werden muss.

---

## Was ist ein Enum

Ein `enum` definiert eine geschlossene Liste möglicher Werte.

```dinoco
enum Role {
	USER
	ADMIN
}
```

In diesem Fall kann `Role` nur `USER` oder `ADMIN` annehmen.

## Verwendung in Models

Nach der Definition kann das Enum als Feldtyp in jedem Model verwendet werden.

```dinoco
enum Role {
	USER
	ADMIN
}

model User {
	id   Integer @id @default(autoincrement())
	role Role    @default(USER)
}
```

Hier:

- `role` verwendet das Enum `Role`.
- `@default(USER)` definiert den Standardwert des Feldes.

## Wann man Enums verwenden sollte

Enums sind nützlich, um Werte wie die folgenden darzustellen:

- Benutzerrollen
- Veröffentlichungsstatus
- Workflow-Schritte
- Zahlungssituationen

Beispiel:

```dinoco
enum PostStatus {
	DRAFT
	REVIEW
	PUBLISHED
	ARCHIVED
}

model Post {
	id     Integer    @id @default(autoincrement())
	title  String
	status PostStatus @default(DRAFT)
}
```

## Best Practices

- Verwenden Sie Enums, wenn die möglichen Werte bekannt und endlich sind.
- Bevorzugen Sie PascalCase-Namen für das Enum und UPPER_CASE-Werte.
- Verwenden Sie `@default(...)`, wenn ein natürlicher Anfangszustand vorhanden ist.
