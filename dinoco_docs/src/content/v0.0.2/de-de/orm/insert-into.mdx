export const title = 'insert_into';

# insert_into

Wird verwendet, um einen Datensatz einzufügen.

---

## Was Sie tun können

- `.values(item)`: definiert den einzufügenden Datensatz.
- `.returning::<T>()`: ändert den Rückgabewert in eine typisierte Projektion des eingefügten Elements.
- `.execute(&client)`: führt den Schreibvorgang in der Datenbank aus.

## Beispiel mit Extend #[insertable]

Verwenden Sie `#[insertable]` zusammen mit `#[derive(Extend)]`, wenn Sie möchten, dass `.values(...)` eine reichhaltigere Nutzlast mit neuen Beziehungen und bestehenden Verbindungen akzeptiert. Dinoco fügt zuerst das übergeordnete Element ein und verarbeitet dann alles rekursiv.

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(User)]
#[insertable]
struct UserWithPosts {
    id: i64,
    name: String,
    posts: Vec<PostWithComments>,
}

#[derive(Debug, Clone, dinoco::Extend)]
#[extend(Post)]
#[insertable]
struct PostWithComments {
    id: i64,
    title: String,
    published: bool,
    authorId: i64,
    comments: Vec<Comment>,
}

dinoco::insert_into::<User>()
    .values(UserWithPosts {
        id: 1,
        name: "Anna".to_string(),
        posts: vec![
            PostWithComments {
                id: 10,
                title: "Rekursiver Insert".to_string(),
                published: true,
                authorId: 0,
                comments: vec![Comment { id: 100, text: "erster".to_string(), flagged: false, postId: 0 }],
            },
        ],
    })
    .execute(&client)
    .await?;
```

## Beispiel mit verschachtelter Beziehung

Verwenden Sie eine `Extend`-Nutzlast mit `#[insertable]`, um das übergeordnete Element zu erstellen und auch das zugehörige Element im selben Fluss zu erstellen.

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(User)]
#[insertable]
struct UserWithProfile {
    id: String,
    username: String,
    role: Role,
    profile: Option<Profile>,
}

dinoco::insert_into::<User>()
    .values(UserWithProfile {
        id: "benutzer-1".to_string(),
        username: "pia".to_string(),
        role: Role::USER,
        profile: Some(Profile {
            id: "profil-1".to_string(),
            bio: Some("Software-Ingenieurin".to_string()),
            userId: String::new(),
        }),
    })
    .execute(&client)
    .await?;
```

## Beispiel mit verschachtelter Verbindung

Wenn die Beziehung auf bereits vorhandene Datensätze verweist, generiert der Codegen ein Enum wie `ArticleConnection`, das innerhalb der Nutzlast verwendet werden kann.

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(Article)]
#[insertable]
struct ArticleWithLabels {
    id: String,
    title: String,
    labels: Vec<ArticleConnection>,
}

dinoco::insert_into::<Article>()
    .values(ArticleWithLabels {
        id: "artikel-10".into(),
        title: "Bestehende verbinden".into(),
        labels: vec![ArticleConnection::Label("label-10".into())],
    })
    .execute(&client)
    .await?;
```

## Beispiel mit ArticleConnection

Dies ist das gängigste Format zum Verbinden bestehender Datensätze in Many-to-Many-Beziehungen.

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(Article)]
#[insertable]
struct ArticleWithLabels {
    id: String,
    title: String,
    labels: Vec<ArticleConnection>,
}

let value = ArticleWithLabels {
    id: "artikel-20".into(),
    title: "Bestehende verbinden".into(),
    labels: vec![
        ArticleConnection::Label("label-10".into()),
        ArticleConnection::Label("label-11".into()),
    ],
};

dinoco::insert_into::<Article>()
    .values(value)
    .execute(&client)
    .await?;
```

## Rückgabe

Ohne `.returning::<T>()` ist die Rückgabe:

```rust
DinocoResult<()>
```

Mit `.returning::<T>()` wird die Rückgabe:

```rust
DinocoResult<T>
```

## Einfaches Beispiel

```rust
dinoco::insert_into::<User>()
    .values(User {
        id: "benutzer_1".to_string(),
        email: "anna@acme.com".to_string(),
        name: "Anna".to_string(),
    })
    .execute(&client)
    .await?;
```

## Beispiel mit typisierter Rückgabe

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(User)]
struct UserSummary {
    id: i64,
    name: String,
}

let created = dinoco::insert_into::<User>()
    .values(User { id: 1, name: "Matthias".to_string() })
    .returning::<UserSummary>()
    .execute(&client)
    .await?;
```
