# update

Used to update filtered records.

---

## What you can do

- `.cond(...)`: defines which records will be updated.
- `.values(item)`: informs the new values for the record.
- `.connect(...)`: creates supported write relation links.
- `.disconnect(...)`: removes supported write relation links.
- `.returning::<T>()`: returns the updated records in a typed projection.
- `.execute(&client)`: executes the update in the database.

## Return

Without `.returning::<T>()`, the return is:

```rust
DinocoResult<()>
```

With `.returning::<T>()`, the return becomes:

```rust
DinocoResult<Vec<T>>
```

Note:

- `update().returning()` does not support relation writes with `.connect(...)` or `.disconnect(...)`.

## Example of field update

```rust
dinoco::update::<User>()
    .cond(|w| w.id.eq(10))
    .values(User {
        id: 10,
        email: "new@acme.com".to_string(),
        name: "New Name".to_string(),
    })
    .execute(&client)
    .await?;
```

## Example with connect(...)

Used to connect relations supported for writing, typically Many to Many.

```rust
dinoco::update::<User>()
    .cond(|w| w.id.eq(10))
    .connect(|r| r.roles().slug.eq("admin"))
    .execute(&client)
    .await?;
```

## Example with disconnect(...)

Used to disconnect relations.

```rust
dinoco::update::<User>()
    .cond(|w| w.id.eq(10))
    .disconnect(|r| r.roles().slug.eq("guest"))
    .execute(&client)
    .await?;
```

## Filters available in connect and disconnect

- `eq`
- `neq`
- `gt`
- `gte`
- `lt`
- `lte`
- `in_values`
- `not_in_values`
- `is_null`
- `is_not_null`
- `includes`
- `starts_with`
- `ends_with`
