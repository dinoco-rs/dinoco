# Конфигурация

Блок `config {}` определяет, как Dinoco будет подключаться к базе данных и какие дополнительные функции будут доступны в проекте.

## Что входит в `config {}`

- `database`: определяет основной адаптер, такой как `sqlite`, `postgresql` или `mysql`.
- `database_url`: указывает основной URL для подключения.
- `read_replicas`: необязательно, добавляет реплики для чтения.
- `redis`: необязательно, включает кэш и очереди на основе Redis.

## Базовый пример

```dinoco
config {
    database = "sqlite"
    database_url = env("DATABASE_URL")
}
```

## Пример с Redis

```dinoco
config {
    database = "postgresql"
    database_url = env("DATABASE_URL")

    redis = {
        url = env("REDIS_URL")
    }
}
```

## Пример с репликами

```dinoco
config {
    database = "postgresql"
    database_url = env("DATABASE_URL")

    read_replicas = [
        env("DATABASE_READ_URL_1"),
        env("DATABASE_READ_URL_2")
    ]
}
```

## Что меняется в сгенерированном API

- С `redis` проект начинает предоставлять вспомогательные функции, такие как `client.cache()`, `.cache(...)` и очереди с `workers()`.
- С `read_replicas` операции чтения могут использовать реплику, в то время как операции записи продолжаются в основной базе данных.
- Без `redis` функции кэширования и очередей не будут полностью доступны.

## Когда редактировать этот блок

- При смене основной базы данных проекта.
- При добавлении Redis для кэширования или очередей.
- При настройке реплики для чтения.
- При настройке переменных среды подключения.
