# 使用方法

`dinoco migrate rollback` コマンドは、最後に適用されたマイグレーションを元に戻します。

データベースへの最新の変更を取り消す必要がある場合に便利です。

---

## コマンドの機能

このコマンドを実行すると、CLI は以下を試みます。

- 最後に適用されたマイグレーションを特定する
- そのマイグレーションを元に戻す
- データベースのマイグレーション履歴を更新する

## パラメータ

[NOME]: マイグレーション名 (特定のマイグレーションに戻す場合) (オプション)

## 使用例

```bash
dinoco migrate rollback
```

## 使用するタイミング

このコマンドは次の場合に使用します。

- 最後のマイグレーションを元に戻す必要がある場合
- スキーマに最近の問題を検出した場合
- 開発環境でデータベースの進化フローを調整している場合

## 注意事項

- すべての構造変更が影響なく元に戻せるほど単純ではありません。
- 重要な環境でロールバックを実行する前に、マイグレーションの影響を確認してください。

## 次のステップ

スキーマを調整した後、新しいマイグレーションを生成できます。

```bash
dinoco migrate generate
```
