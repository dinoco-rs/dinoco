# 使用方法

`dinoco migrate rollback` コマンドは、最後に適用されたマイグレーションを元に戻します。

データベースの最新の変更を取り消す必要がある場合に便利です。

---

## コマンドの機能

このコマンドを実行すると、CLIは以下を試みます。

- 最後に適用されたマイグレーションを特定する
- そのマイグレーションを元に戻す
- データベースのマイグレーション履歴を更新する

## パラメータ

[名前]: マイグレーションの名前（特定のマイグレーションにロールバックする場合） (オプション)

## 使用例

```bash
dinoco migrate rollback
```

## 使用するタイミング

このコマンドは以下の場合に使用します。

- 最後のマイグレーションを元に戻す必要がある場合
- スキーマに最近問題が検出された場合
- 開発中にデータベースの進化フローを調整している場合

## 注意事項

- すべての構造変更が影響なく元に戻せるわけではありません。
- 重要な環境でロールバックを実行する前に、マイグレーションの影響を確認してください。

## 次のステップ

スキーマを調整した後、新しいマイグレーションを生成できます。

```bash
dinoco migrate generate
```
