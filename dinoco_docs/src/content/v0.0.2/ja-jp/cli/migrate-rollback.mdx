# 使い方

`dinoco migrate rollback` コマンドは、最後に適用された migration を取り消します。

データベース内の直近の変更を取り消す必要があるときに便利です。

---

## コマンドの動作

このコマンドを実行すると、CLI は次を試みます:

- 最後に適用されたマイグレーションを特定する
- このマイグレーションを元に戻す
- データベースの migration 履歴を更新する

## パラメータ

[NAME]: マイグレーションの名前（特定のマイグレーションに戻す場合） (オプション)

## 使用例

```bash
dinoco migrate rollback
```

## 使うタイミング

次のようなときにこのコマンドを使います:

- 最後のマイグレーションを元に戻す必要がある場合
- スキーマに最近の問題を発見した場合
- 開発中のデータベース進化フローを調整する

## 注意点

- すべての構造変更が、影響なく簡単に元に戻せるわけではありません。
- 重要な環境でロールバックを実行する前に、マイグレーションの影響を確認してください。

## 次のステップ

スキーマを調整した後、新しいマイグレーションを生成できます。

```bash
dinoco migrate generate
```
