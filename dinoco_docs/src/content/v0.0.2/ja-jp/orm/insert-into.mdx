export const title = 'insert_into';

# insert_into

レコードを挿入するために使用されます。

---

## できること

- `.values(item)`: 挿入するレコードを定義します。
- `.returning::&lt;T&gt;()`: 戻り値を、挿入されたアイテムの型付きプロジェクションに変更します。
- `.execute(&client)`: データベースへの書き込みを実行します。

## Extend #[insertable] の例

#[insertable] を #[derive(Extend)] と一緒に使用すると、.values(...) が新しいリレーションと既存の接続を含むよりリッチなペイロードを受け入れるようになります。Dinoco はまず親を挿入し、その後すべてを再帰的に処理します。

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(User)]
#[insertable]
struct UserWithPosts {
    id: i64,
    name: String,
    posts: Vec<PostWithComments>,
}

#[derive(Debug, Clone, dinoco::Extend)]
#[extend(Post)]
#[insertable]
struct PostWithComments {
    id: i64,
    title: String,
    published: bool,
    authorId: i64,
    comments: Vec<Comment>,
}

dinoco::insert_into::<User>()
    .values(UserWithPosts {
        id: 1,
        name: "アナ".to_string(),
        posts: vec![
            PostWithComments {
                id: 10,
                title: "再帰的な挿入".to_string(),
                published: true,
                authorId: 0,
                comments: vec![Comment { id: 100, text: "最初".to_string(), flagged: false, postId: 0 }],
            },
        ],
    })
    .execute(&client)
    .await?;
```

## ネストされたリレーションの例

#[insertable] を持つ Extend ペイロードを使用して、親を作成し、同じフローで関連するレコードも作成します。

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(User)]
#[insertable]
struct UserWithProfile {
    id: String,
    username: String,
    role: Role,
    profile: Option<Profile>,
}

dinoco::insert_into::<User>()
    .values(UserWithProfile {
        id: "user-1".to_string(),
        username: "ビア".to_string(),
        role: Role::USER,
        profile: Some(Profile {
            id: "profile-1".to_string(),
            bio: Some("ソフトウェアエンジニア".to_string()),
            userId: String::new(),
        }),
    })
    .execute(&client)
    .await?;
```

## ネストされた接続の例

リレーションが既存のレコードを指す場合、codegen はペイロード内で使用される ArticleConnection のような enum を生成します。

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(Article)]
#[insertable]
struct ArticleWithLabels {
    id: String,
    title: String,
    labels: Vec<ArticleConnection>,
}

dinoco::insert_into::<Article>()
    .values(ArticleWithLabels {
        id: "article-10".into(),
        title: "既存の接続".into(),
        labels: vec![ArticleConnection::Label("label-10".into())],
    })
    .execute(&client)
    .await?;
```

## ArticleConnection の例

これは、多対多のリレーションにおける既存レコードの接続で最も一般的な形式です。

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(Article)]
#[insertable]
struct ArticleWithLabels {
    id: String,
    title: String,
    labels: Vec<ArticleConnection>,
}

let value = ArticleWithLabels {
    id: "article-20".into(),
    title: "既存の接続".into(),
    labels: vec![
        ArticleConnection::Label("label-10".into()),
        ArticleConnection::Label("label-11".into()),
    ],
};

dinoco::insert_into::<Article>()
    .values(value)
    .execute(&client)
    .await?;
```

## 戻り値

`.returning::&lt;T&gt;()` なしの場合、戻り値は次のようになります。

```rust
DinocoResult<()>
```

`.returning::&lt;T&gt;()` ありの場合、戻り値は次のようになります。

```rust
DinocoResult<T>
```

## シンプルな例

```rust
dinoco::insert_into::<User>()
    .values(User {
        id: "usr_1".to_string(),
        email: "アナ@acme.com".to_string(),
        name: "アナ".to_string(),
    })
    .execute(&client)
    .await?;
```

## 型付き戻り値の例

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(User)]
struct UserSummary {
    id: i64,
    name: String,
}

let created = dinoco::insert_into::<User>()
    .values(User { id: 1, name: "マテウス".to_string() })
    .returning::<UserSummary>()
    .execute(&client)
    .await?;
```

## ワーカーの例

```rust
use database::*;

let _worker = workers()
    .on::<User, _, _>("user.created", |job| async move {
        println!("ユーザーが作成されました: {}", job.data.name);
        job.success();
    })
    .run()
    .await?;

dinoco::insert_into::<User>()
    .values(User { id: 1, name: "マテウス".to_string() })
    .enqueue("user.created")
    .execute(&client)
    .await?;
```

ワーカーの詳細については、[**`queues`**](/v0.0.2/orm/queues) を参照してください。
