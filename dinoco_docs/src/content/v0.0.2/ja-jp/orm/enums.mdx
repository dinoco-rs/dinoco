# Enum

Enumを使用すると、Dinocoスキーマ内のフィールドを既知の固定値のセットに制限できます。

値が予測可能で、検証され、モデル間で再利用される必要がある場合に便利です。

---

## Enumとは

`enum` は、可能な値の閉じたリストを定義します。

```dinoco
enum Role {
	USER
	ADMIN
}
```

この場合、`Role` は `USER` または `ADMIN` のいずれかのみを取ることができます。

## モデルでの使用

定義後、enumは任意のモデルのフィールド型として使用できます。

```dinoco
enum Role {
	USER
	ADMIN
}

model User {
	id   Integer @id @default(autoincrement())
	role Role    @default(USER)
}
```

ここでは：

- `role` は `Role` enumを使用します。
- `@default(USER)` はフィールドのデフォルト値を定義します。

## Enumを使用するタイミング

Enumは次のような値を表現するのに役立ちます：

- ユーザーの役割
- 投稿ステータス
- ワークフローの段階
- 支払い状況

例：

```dinoco
enum PostStatus {
	DRAFT
	REVIEW
	PUBLISHED
	ARCHIVED
}

model Post {
	id     Integer    @id @default(autoincrement())
	title  String
	status PostStatus @default(DRAFT)
}
```

## ベストプラクティス

- 可能な値が既知で有限である場合にenumを使用します。
- enumにはPascalCase、値にはUPPER_CASEの名前を推奨します。
- 自然な初期状態がある場合は `@default(...)` を使用します。
