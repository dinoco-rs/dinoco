# 列挙型

列挙型を使用すると、Dinocoスキーマ内のフィールドを、既知の固定された値のセットに制限できます。

値が予測可能で、検証済みであり、モデル間で再利用する必要がある場合に役立ちます。

---

## Enum とは

`enum` は、取り得る値の閉じたリストを定義します。

```dinoco
enum Role {
	USER
	ADMIN
}
```

この場合、`Role` は `USER` または `ADMIN` のいずれかの値のみを取ることができます。

## モデルでの使用

定義後、列挙型は任意のモデルでフィールド型として使用できます。

```dinoco
enum Role {
	USER
	ADMIN
}

model User {
	id   Integer @id @default(autoincrement())
	role Role    @default(USER)
}
```

ここで:

- `role` は列挙型 `Role` を使用します。
- `@default(USER)` はフィールドのデフォルト値を定義します。

## Enum を使うタイミング

列挙型は、次のような値を表現するのに役立ちます。

- ユーザーロール
- 公開ステータス
- ワークフローステップ
- 支払い状況

例:

```dinoco
enum PostStatus {
	DRAFT
	REVIEW
	PUBLISHED
	ARCHIVED
}

model Post {
	id     Integer    @id @default(autoincrement())
	title  String
	status PostStatus @default(DRAFT)
}
```

## ベストプラクティス

- 取り得る値が既知で有限である場合は、列挙型を使用してください。
- 列挙型にはPascalCase、値にはUPPER_CASEの名前を推奨します。
- 自然な初期状態がある場合は、`@default(...)` を使用してください。
