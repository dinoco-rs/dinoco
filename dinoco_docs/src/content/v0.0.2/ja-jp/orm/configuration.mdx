# 設定

`config {}`ブロックは、Dinocoがデータベースに接続する方法と、プロジェクトで利用できる追加機能について定義します。

## `config {}`に含まれるもの

- `database`: `sqlite`、`postgresql`、`mysql`などの主要なアダプターを定義します。
- `database_url`: 主要な接続URLを指定します。
- `read_replicas`: オプションで、読み取りレプリカを追加します。
- `redis`: オプションで、Redisベースのキャッシュとキューを有効にします。

## 基本例

```dinoco
config {
    database = "sqlite"
    database_url = env("DATABASE_URL")
}
```

## Redisを使用した例

```dinoco
config {
    database = "postgresql"
    database_url = env("DATABASE_URL")

    redis = {
        url = env("REDIS_URL")
    }
}
```

## レプリカを使用した例

```dinoco
config {
    database = "postgresql"
    database_url = env("DATABASE_URL")

    read_replicas = [
        env("DATABASE_READ_URL_1"),
        env("DATABASE_READ_URL_2")
    ]
}
```

## 生成されたAPIの変更点

- `redis`を使用すると、プロジェクトは`client.cache()`、`.cache(...)`などの補助関数や、`workers()`を使用したキューを公開するようになります。
- `read_replicas`を使用すると、読み取りはレプリカを使用でき、書き込みは引き続きメインデータベースで行われます。
- `redis`がない場合、キャッシュおよびキュー機能は完全に利用できません。

## このブロックを編集するタイミング

- プロジェクトのメインデータベースを変更する場合。
- キャッシュまたはキューのためにRedisを追加する場合。
- 読み取りレプリカを設定する場合。
- 接続環境変数を調整する場合。
