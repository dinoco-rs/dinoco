# Dinoco 소개

이 라이브러리는 데이터베이스 관리를 용이하게 하기 위해 만들어졌으며, [Prisma](https://www.prisma.io/)의 철학에서 영감을 받아 Dinoco는 [Rust](https://www.rust-lang.org/) 생태계에 유연하고 타입이 지정되며 매우 생산적인 인터페이스를 제공합니다.

---

## Rust에서 Prisma 경험

Dinoco는 Prisma의 민첩성과 강력한 타입 지정을 [Rust](https://www.rust-lang.org/) 생태계로 가져오려는 열망에서 탄생했습니다. 우리는 일상적인 작업을 위해 복잡한 매크로를 구성하거나 수동으로 SQL을 작성하는 데 시간을 낭비해서는 안 된다고 믿습니다.

## 프로젝트의 기둥

- **DX (개발자 경험) 우선:** 인지 부하를 줄이고 개발 속도를 높이는 유연하고 직관적인 API.
- **종단 간 타입 안전성:** 데이터가 항상 타입 정의와 동기화되어 컴파일 시점에 오류를 포착하도록 보장합니다.
- **간단한 관계:** 번거로운 수동 조인을 잊으세요. 관계를 정의하면 Dinoco가 내부적으로 복잡성을 해결합니다.
- **강력한 CLI:** 워크플로우를 체계적이고 빠르게 유지하기 위한 통합된 마이그레이션 관리 및 코드 생성.

## 우리의 위치는 어디인가요?

[Rust](https://www.rust-lang.org/) 생태계는 강력한 도구를 가지고 있지만, 종종 가파른 학습 곡선을 요구합니다. **Dinoco**는 현재 옵션들 사이의 간극을 메우기 위해 등장했습니다:

- **[Diesel](https://diesel.rs/)과 비교하여:** Diesel은 SQL에 대한 깊은 지식과 복잡한 DSL(Domain Specific Language)을 요구하는 반면, Dinoco는 간단한 쿼리에 대해 컴파일러와 씨름할 필요 없이 생산성에 중점을 둔 깔끔한 구문을 우선시합니다.
- **[SeaORM](https://www.sea-ql.org/SeaORM/)과 비교하여:** SeaORM은 훌륭하지만, 계층 기반 아키텍처는 많은 프로젝트에 대해 지나치게 장황할 수 있습니다. Dinoco는 불필요한 "상용구"를 제거하여 훨씬 적은 코드 라인으로 동일한 견고성을 제공합니다.

[Rust](https://www.rust-lang.org/)의 성능과 Prisma가 Node.js 세계에 가져온 유연성을 찾고 있다면, Dinoco는 다음 프로젝트를 위한 올바른 선택입니다.
