# 파생

Dinoco에서 사용되는 핵심 파생 특성 및 속성에 대한 개요입니다.

---

## Dinoco의 파생 특성이란 무엇인가요?

Dinoco의 파생 특성은 Rust 구조체를 ORM 동작에 연결하여 전체 구현을 수동으로 작성할 필요 없이 도와줍니다.

일상에서 가장 일반적으로 사용되는 것은 다음과 같습니다:

- `#[derive(Rowable)]`
- `#[derive(Extend)]`
- `#[insertable]`

## Rowable 파생 특성

구조체가 Dinoco에 의해 직접 읽고 쓰여질 모델을 나타낼 때 `Rowable`을 사용하세요.

```rust
#[derive(Debug, Clone, dinoco::Rowable)]
struct User {
    id: String,
    email: String,
    name: String,
}
```

이 파생 특성은 어댑터에서 반환된 행을 Rust 구조체로 직렬화합니다.

## Extend 파생 특성

기본 모델과 다른 프로젝션이 필요할 때, 일반적으로 `select`, `include`, `count` 또는 풍부한 페이로드에 대해 `Extend`를 사용하세요.

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(User)]
struct UserSummary {
    id: String,
    name: String,
}
```

이 경우, 구조체는 `User` 모델에 계속 연결되지만, 해당 흐름에 필요한 필드만 노출할 수 있습니다.

## #[insertable] 속성

`.values(...)`가 새로운 관계 또는 페이로드 내의 기존 연결을 받아들여야 할 때 `Extend`와 함께 `#[insertable]`을 사용하세요.

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(Article)]
#[insertable]
struct ArticleWithLabels {
    id: String,
    title: String,
    labels: Vec<ArticleConnection>,
}
```

이를 통해 Dinoco는 부모를 삽입한 다음 중첩된 항목을 자동으로 처리합니다.

## Rowable 예시

```rust
#[derive(Debug, Clone, dinoco::Rowable)]
struct Team {
    id: String,
    name: String,
}

dinoco::insert_into::<Team>()
    .values(Team {
        id: "team-1".into(),
        name: "Platform".into(),
    })
    .execute(&client)
    .await?;
```

## Extend 예시

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(User)]
struct UserWithPostsCount {
    id: String,
    name: String,
    posts_count: usize,
}

let users = dinoco::find_many::<User>()
    .select::<UserWithPostsCount>()
    .count(|user| user.posts())
    .execute(&client)
    .await?;
```

## #[insertable] 예시

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(Article)]
#[insertable]
struct ArticleWithLabels {
    id: String,
    title: String,
    labels: Vec<ArticleConnection>,
}

dinoco::insert_many::<Article>()
    .values(vec![
        ArticleWithLabels {
            id: "article-11".into(),
            title: "Connect Multiple".into(),
            labels: vec![
                ArticleConnection::Label("label-11".into()),
                ArticleConnection::Label("label-12".into()),
            ],
        },
        ArticleWithLabels {
            id: "article-12".into(),
            title: "Connect Batch".into(),
            labels: vec![ArticleConnection::Label("label-10".into())],
        },
    ])
    .execute(&client)
    .await?;
```

## 각각 언제 사용해야 하나요?

- 생성된 주 모델 또는 직접 행 읽기와 호환되는 구조체에 `Rowable`을 사용하세요.
- 프로젝션, 카운트, 포함(includes) 및 특수 페이로드에 `Extend`를 사용하세요.
- 중첩된 쓰기 페이로드로도 `Extend`가 사용될 때 `#[insertable]`을 사용하세요.
