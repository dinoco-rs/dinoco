# 사용 방법

명령어 `dinoco migrate rollback`는 마지막으로 적용된 마이그레이션을 되돌립니다.

데이터베이스에서 가장 최근 변경 사항을 취소해야 할 때 유용합니다.

---

## 명령어가 하는 일

이 명령어를 실행하면 CLI는 다음을 시도합니다:

- 마지막으로 적용된 마이그레이션 식별
- 해당 마이그레이션 되돌리기
- 데이터베이스 마이그레이션 기록 업데이트

## 매개변수

[이름]: 마이그레이션 이름 (특정 마이그레이션으로 되돌리려면) (선택 사항)

## 사용 예시

```bash
dinoco migrate rollback
```

## 사용 시점

다음과 같은 경우 이 명령어를 사용하세요:

- 마지막 마이그레이션을 되돌려야 할 때
- 스키마에서 최근 문제를 발견했을 때
- 개발 중 데이터베이스 진화 흐름을 조정하고 있을 때

## 주의 사항

- 모든 구조적 변경이 영향 없이 되돌리기 쉬운 것은 아닙니다.
- 중요한 환경에서 롤백을 수행하기 전에 마이그레이션의 영향을 검토하세요.

## 다음 단계

스키마를 조정한 후 새 마이그레이션을 생성할 수 있습니다:

```bash
dinoco migrate generate
```
