# insert_into

Usato per inserire un record.

---

## Cosa puoi fare

- `.values(item)`: definisce il record che verrà inserito.
- `.returning::\<T\>()`: cambia il ritorno a una proiezione tipizzata dell'elemento inserito.
- `.execute(&client)`: esegue la scrittura nel database.

## Esempio con Extend #[insertable]

Usa `#[insertable]` insieme a `#[derive(Extend)]` quando vuoi che `.values(...)` accetti un payload più ricco con nuove relazioni e connessioni esistenti. Dinoco inserisce prima il padre e poi elabora tutto in modo ricorsivo.

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(User)]
#[insertable]
struct UserWithPosts {
    id: i64,
    name: String,
    posts: Vec<PostWithComments>,
}

#[derive(Debug, Clone, dinoco::Extend)]
#[extend(Post)]
#[insertable]
struct PostWithComments {
    id: i64,
    title: String,
    published: bool,
    authorId: i64,
    comments: Vec<Comment>,
}

dinoco::insert_into::<User>()
    .values(UserWithPosts {
        id: 1,
        name: "Anna".to_string(),
        posts: vec![
            PostWithComments {
                id: 10,
                title: "Inserimento ricorsivo".to_string(),
                published: true,
                authorId: 0,
                comments: vec![Comment { id: 100, text: "primo".to_string(), flagged: false, postId: 0 }],
            },
        ],
    })
    .execute(&client)
    .await?;
```

## Esempio con relazione annidata

Usa un payload `Extend` con `#[insertable]` per creare il padre e anche creare il correlato nello stesso flusso.

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(User)]
#[insertable]
struct UserWithProfile {
    id: String,
    username: String,
    role: Role,
    profile: Option<Profile>,
}

dinoco::insert_into::<User>()
    .values(UserWithProfile {
        id: "user-1".to_string(),
        username: "bia".to_string(),
        role: Role::USER,
        profile: Some(Profile {
            id: "profile-1".to_string(),
            bio: Some("Ingegnere del software".to_string()),
            userId: String::new(),
        }),
    })
    .execute(&client)
    .await?;
```

## Esempio con connessione annidata

Quando la relazione punta a record già esistenti, il codegen genera un enum come `ArticleConnection` da usare all'interno del payload.

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(Article)]
#[insertable]
struct ArticleWithLabels {
    id: String,
    title: String,
    labels: Vec<ArticleConnection>,
}

dinoco::insert_into::<Article>()
    .values(ArticleWithLabels {
        id: "article-10".into(),
        title: "Connetti Esistente".into(),
        labels: vec![ArticleConnection::Label("label-10".into())],
    })
    .execute(&client)
    .await?;
```

## Esempio con ArticleConnection

Questo è il formato più comune per la connessione di record esistenti in relazioni many-to-many.

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(Article)]
#[insertable]
struct ArticleWithLabels {
    id: String,
    title: String,
    labels: Vec<ArticleConnection>,
}

let value = ArticleWithLabels {
    id: "article-20".into(),
    title: "Connetti Esistente".into(),
    labels: vec![
        ArticleConnection::Label("label-10".into()),
        ArticleConnection::Label("label-11".into()),
    ],
};

dinoco::insert_into::<Article>()
    .values(value)
    .execute(&client)
    .await?;
```

## Ritorno

Senza `.returning::\<T\>()`, il ritorno è:

```rust
DinocoResult<()>
```

Con `.returning::\<T\>()`, il ritorno diventa:

```rust
DinocoResult<T>
```

## Esempio semplice

```rust
dinoco::insert_into::<User>()
    .values(User {
        id: "usr_1".to_string(),
        email: "anna@acme.com".to_string(),
        name: "Anna".to_string(),
    })
    .execute(&client)
    .await?;
```

## Esempio con ritorno tipizzato

```rust
#[derive(Debug, Clone, dinoco::Extend)]
#[extend(User)]
struct UserSummary {
    id: i64,
    name: String,
}

let created = dinoco::insert_into::<User>()
    .values(User { id: 1, name: "Matteo".to_string() })
    .returning::<UserSummary>()
    .execute(&client)
    .await?;
```

## Esempio con worker

```rust
use database::*;

let _worker = workers()
    .on::<User, _, _>("user.created", |job| async move {
        println!("Utente creato: {}", job.data.name);
        job.success();
    })
    .run()
    .await?;

dinoco::insert_into::<User>()
    .values(User { id: 1, name: "Matteo".to_string() })
    .enqueue("user.created")
    .execute(&client)
    .await?;
```
